# 수집형 RPG 설계 원칙 바이블

> 수집형 RPG 성공 25종 + 실패/종료 25종 = 총 50종 분석 + 설계 원칙 조사에서 귀납적으로 도출한 설계 원칙
> 구체적 수치/사례가 필요하면 `research/collection-rpg/` 폴더의 원시 데이터를 참조한다

## 원칙 0. 수집형 RPG의 본질 — "이 캐릭터를 갖고 싶다"가 모든 것의 출발점

> 수집형 RPG에서 모든 시스템(전투/성장/BM/소셜)은 "이 캐릭터를 갖고 싶다"라는 욕구를 강화하는 방향으로 설계되어야 한다.

50종 분석 결론: **캐릭터를 갖고 싶지 않으면 가챠도, 성장도, BM도 작동하지 않는다.**

"갖고 싶다"를 만드는 5가지 요소 (우선순위 순):

| 순위 | 요소 | 설명 | 데이터 |
|------|------|------|--------|
| 1 | **비주얼** | 디자인, 전투 이펙트, 일러스트 | 픽업 의사결정의 최초 트리거 |
| 2 | **성능** | 강한 캐릭터 = 콘텐츠 클리어 | FOMO 유발의 핵심 |
| 3 | **스토리/세계관** | 캐릭터 개인 퀘스트, 관계성 | 감정적 투자 → 장기 충성 |
| 4 | **보이스** | 고품질 성우 퍼포먼스 | 일본/한국 시장에서 결정적 |
| 5 | **한정성** | "지금 아니면 못 얻음" | FOMO → 과금 트리거 |

## 원칙 1. 전투 시스템 — "모바일에서 최초"가 $5B 이상의 조건

### 데이터

$5B 이상 달성한 수집형 RPG 4종의 공통점:

| 게임 | 전투 메커닉 | 수익 | "최초" 여부 |
|------|-----------|------|-----------|
| Monster Strike | 핀볼 물리+RPG | $11.4B | 모바일 최초 |
| Puzzle & Dragons | 매치3 퍼즐+RPG | $7B | 모바일 최초 |
| Fate/Grand Order | 턴제 카드+IP 스토리 | $7B | IP 충성도 예외 |
| Genshin Impact | 오픈월드 액션+가챠 | $6B+ | 모바일 최초 |

**법칙**: $5B+ 메가히트는 전부 "모바일에서 이 전투를 최초로 한 게임"이다. FGO만이 IP 충성도로 예외.

### 전투 유형별 현황 (50종 기준)

| 전투 유형 | 성공작 비율 | 특징 |
|-----------|-----------|------|
| 턴제 커맨드 | 높음 (7/25 성공작) | 가장 안전한 선택, 그러나 차별화 어려움 |
| 실시간 액션 | 높음 (5/25) | 콘솔급 요구 → 개발 비용 최고 |
| 턴제/실시간 자동 | 중간 (4/25) | 캐주얼 접근성 좋음, 방치형 결합 용이 |
| 퍼즐+RPG | 높음 (2/25, 그러나 $14B) | P&D와 Dokkan이 장르 독점 |
| 타워디펜스 | 1종이 $1B+ | Arknights 유일 → 장르 포지셔닝의 힘 |
| 물리 기반 | 1종이 $11B+ | Monster Strike 유일 → 독창성의 극한 보상 |

### "세미-오토"가 2024 트렌드

자동 전투 + 전략적 배치/타이밍 = 접근성과 깊이를 동시에 잡는 현재 표준.
- AFK Journey: 자동 전투 + 진영 배치
- NIKKE: 자동 사격 + 스킬 타이밍
- Blue Archive: 자동 전투 + 스킬 발동 수동

### 전투 세션 길이

- 1전투 적정: **2~5분**
- 턴제: 5~10분, 자동: 1~3분
- 상위 25% 게임 평균 세션: 8~9분, 하루 평균 4세션

### 체크리스트

```
□ 전투 시스템이 "모바일에서 최초"인 요소를 포함하는가?
□ 전투에서 수집한 캐릭터의 개성이 느껴지는가?
□ 1전투 2~5분 범위인가?
□ 자동+수동의 밸런스가 설계되어 있는가? (세미-오토)
□ 동일 전투 방식의 선행 성공작이 이미 시장을 점령하고 있지 않은가? (법칙 19)
```

## 원칙 2. 캐릭터 설계 — 출시 20~50종, 등급 3~5성, 개성이 핵심

### 출시 시 캐릭터 수

| 게임 | 출시 시 | 현재 |
|------|--------|------|
| Genshin Impact | ~22종 | 100종+ |
| Honkai: Star Rail | ~20종 | 80종+ |
| Arknights | ~50종 | 300종+ |

**20~50종이 적정**. 핵심은 수량이 아니라 **각 캐릭터의 전투 메커니즘 차별화**.

### 등급 체계

- **3~5성(3단계)이 HoYoverse 이후 업계 디폴트**
- 최고 등급 확률: **0.6~3%** (Genshin 0.6%, Arknights 2%, Blue Archive 3%)
- 등급 간 차이 = 수치가 아닌 **메커니즘 변화** (rpg-skill-design-bible.md 원칙과 동일)

### 캐릭터 역할 — 3분류를 넘어서

기본 탱커/딜러/서포터에 추가:

| 역할 | 설명 | 효과 |
|------|------|------|
| 브레이커 | 적 방어/상태 파괴 | 팀 시너지 다양화 |
| 버퍼/디버퍼 | 아군 강화/적 약화 | 서포터 세분화 |
| 서브 DPS | 보조 딜러, 원소 반응 등 | 조합 깊이 |

**역할 분류보다 역할 간 시너지가 더 중요**: "최소 5가지 팀 조합이 실용적으로 성립"해야 메타 고착 방지.

### 파워 인플레이션(Power Creep) 대응

실패 사례:
- FFBE: $8.96억 → 2024년 $990만으로 붕괴, 서비스 종료
- Valkyrie Crusade: 파워 크리프 → 고래만 남다가 종료

대응 전략:
1. **수직 파워업 대신 수평 다양성** — 신규 캐릭터 = 기존 대체가 아닌 새 메커니즘
2. **콘텐츠 특화 메타** — 구형 캐릭터도 특정 콘텐츠에서 빛나는 설계
3. **정기 리밸런싱** — 버프/너프로 메타 순환

### 중복(Duplicate) 처리

- **Best Practice**: 중복 = "강화 재료"로 전환 (Genshin 별자리, HSR 에이돌론)
- **Worst**: 중복 = 그냥 소멸 → 유저 분노
- **중요**: 중복이 "낭비"로 느껴지면 유저 불만 폭발

### 체크리스트

```
□ 출시 시 캐릭터 20~50종 범위인가?
□ 등급이 3~5단계인가?
□ 각 캐릭터의 전투 메커니즘이 차별화되어 있는가?
□ 실용적 팀 조합이 5가지 이상 성립하는가?
□ 파워 인플레이션 방지 전략이 설계되어 있는가?
□ 중복 캐릭터가 "강화 재료"로 전환되는가?
```

## 원칙 3. 가챠/천장 — 천장 없으면 3년 내 종료 확률 70%

### 핵심 데이터

- 2,200개 가챠 타이틀 중 **70%(1,541개)가 3년 내 종료**
- 1년 내 종료: 501개, 2년 내 추가: 554개
- **천장 없이 장기 성공 = FGO 단 하나** (IP 충성도 극강의 예외)
- 천장 도입 시 역설적으로 **평균 지출 증가** ("안전하다" → 지출 한도 상향)

### 천장 설계 표준 (2024 기준)

```
[소프트 피티] 50~75연 — 확률 점진 증가 시작
[하드 피티] 80~100연 — 최고 등급 100% 보장
[50/50 보장] 하드 피티 2회분 — 픽업 실패 시 다음 100% 확정
[교환/Spark] 150~300연 — 포인트 누적 → 직접 선택 구매
```

### 천장까지 비용

- **$150~$200** 수준이 업계 표준
- 월정액($5) 유저가 2~3개월이면 천장 도달하는 설계가 이상적

### 체크리스트

```
□ 천장 시스템이 설계되어 있는가? (신규 게임은 필수)
□ 소프트 피티 + 하드 피티 + 보장의 다층 구조인가?
□ 천장까지 비용이 $150~200 범위인가?
□ 한국/중국/일본 확률 공개 법규를 준수하는가?
```

## 원칙 4. 성장 시스템 — 4~5층, 순차 해금, 벽은 "기회"로

### 표준 성장 레이어

```
[즉시 루프 — 5~15분]
캐릭터 레벨업 → 스킬 레벨업

[중기 루프 — 일/주 단위]
돌파/각성 → 장비 세팅 → 팀 조합 최적화

[장기 루프 — 월 이상]
최고 등급 달성 → 특화 강화 → PvP 메타 → 길드
```

### 적정 레이어 수

- **4~5층이 표준**
- 3층 이하: 장기 동기 부족, BM 연결점 부족
- 6층 이상: 유저 압도, 운영 비용 폭증 (Genshin/RAID만 감당 가능)
- **순차 해금 필수** — Day 1에 전부 공개 금지

### 벽(Wall) 설계

벽은 "좌절"이 아니라 **"목표가 명확한 과금 기회"**:

```
[좋은 벽] "이 재료 3일 모으면 돌파 가능. 지금 과금하면 오늘 가능"
[나쁜 벽] "모르겠고 어쨌든 진행 불가. 뭘 해야 하는지도 불명확"
```

### 무과금 vs 과금 격차

| 게임 유형 | 적정 격차 |
|----------|---------|
| PvE 중심 | 3~6개월 (무과금도 도달 가능, 시간 차이만) |
| PvP 포함 | 1~3개월 (과도한 격차 = 무과금 이탈 = 생태계 붕괴) |

### 체크리스트

```
□ 성장 레이어가 4~5층인가?
□ 순차적으로 해금되는가? (Day 1 전부 공개 아닌가?)
□ 벽이 "목표 명확한 기회"인가? ("좌절"이 아닌가?)
□ 무과금/과금 격차가 장르 적정 범위인가?
```

## 원칙 5. 라이브서비스 — 6주 주기, 70%가 3년 내 사망

### 업데이트 주기 표준

| 유형 | 주기 | 비고 |
|------|------|------|
| 메이저 패치 (신캐릭+스토리) | **6주** | HoYoverse 이후 업계 디폴트 |
| 마이너 이벤트 | **1~2주** | |
| 시즌 이벤트 | **30~45일** | 67% 상위권 타이틀 |

### 70% 3년 내 종료의 의미

| 구간 | 종료 수 | 누적 비율 |
|------|--------|---------|
| 1년 내 | 501개 | 31.7% |
| 2년 내 | +554개 | 56.8% |
| 3년 내 | +295개 | **70%** |

**근본 원인: 라이브서비스 속도를 따라갈 인력/자금 부재.** 법칙 16과 동일.

### 소규모 팀의 라이브서비스 생존 전략

1. **방치형 요소** — 유저가 매일 직접 플레이하지 않아도 진행 → 운영 부담 감소
2. **로그라이트 요소** — 랜덤 조합이 콘텐츠 자동 생성 (Capybara Go 전략)
3. **모듈형 이벤트** — 골격 재활용, 보상만 변경
4. **유저 생성 메타** — 팀 조합/공략을 커뮤니티가 만들어냄

### 체크리스트

```
□ 6주 주기 메이저 패치를 감당할 팀 규모인가?
□ 감당 못 한다면, 라이브 부담을 줄이는 구조(방치/로그라이트/모듈형)가 있는가?
□ 3년 운영 계획이 수립되어 있는가?
```

## 원칙 8. 소규모 팀의 수집형 RPG — 법칙 16을 넘는 방법

### 법칙 16의 경고

> "소형팀의 가챠 라이브서비스는 물리적으로 지속 불가능한 구조"

### 법칙 16을 우회하는 3가지 구조

| 구조 | 원리 | 검증 사례 |
|------|------|---------|
| **방치형 코어** | 콘텐츠 소비 속도 자체를 늦춤 | AFK Arena $1.08B, Memento Mori $40M 첫 달 |
| **로그라이트 코어** | 랜덤 조합 = 콘텐츠 자동 생성 | Capybara Go $1.09억 |
| **건설/경영 메타** | 전투 외 콘텐츠로 시간 분산 | Cookie Run: Kingdom $220M+ |

**핵심**: 이 3가지 중 최소 1개를 코어에 내장해야 소규모 팀이 수집형 RPG를 운영할 수 있다.

### 체크리스트

```
□ 10~15명 팀이라면, 라이브 부담 감소 구조 3가지 중 최소 1개가 있는가?
□ 6주 주기 신규 캐릭터를 팀이 감당할 수 있는가?
□ 감당 못 한다면 "캐릭터 없이도 콘텐츠가 갱신되는 구조"인가?
```
