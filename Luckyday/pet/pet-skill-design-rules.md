# 펫 스킬 설계 규칙

> 운수좋은날 펫 스킬 기획 시 반드시 따라야 하는 규칙 문서
> 카탈로그(`pet-skill-catalog.md`)와 함께 사용

---

## 1. 펫의 역할 정의

펫은 **두두의 서포터**임. 전투의 주체는 두두 + 슬롯이고, 펫은 이를 보조함.

- 펫은 직접 전투하지 않음
- 펫의 가치는 **"어떤 전략을 가능하게 하느냐"**로 결정됨
- 유저의 핵심 의사결정: **전투 전에 어떤 펫 2마리를 편성할 것인가**

---

## 2. 펫이 할 수 없는 것 (절대 규칙)

| 제한 | 이유 |
|---|---|
| ❌ 직접 데미지 (공격력 N%로 공격) | 추억장식 곱배율로 스핀 데미지가 뻥튀기되므로 펫 직접 데미지는 무의미 |
| ❌ 화상 직접 부여 | 화상 = 해당 턴 화염 최종 데미지의 10%. 펫에게는 기준이 될 화염 데미지가 없음 |
| ❌ 출혈 직접 부여 | 출혈 = 해당 턴 자연 최종 데미지의 10%. 동일한 이유 |

---

## 3. 펫이 할 수 있는 것

| 가능 | 예시 |
|---|---|
| ✅ 저체온 부여 | 스택 기반. 3스택 → 빙결 1턴 |
| ✅ 어지러움 부여 | 스택 기반. 1스택당 적 공격력 -10%, 빗맞음 확률 +10% |
| ✅ 두두 버프 | 공격력, 크리, 속성 데미지, 피해 증폭 등 |
| ✅ 보호/생존 | 보호막, 회복, 재생, 방패, 정화, 면역 |
| ✅ 슬롯 조작 | 부적, 조커 심볼, 반반심볼, x2룬, 심볼 변환, 홀드 등 |
| ✅ 적 디버프 | 약화(받는 피해 증가), 버프 해제 |
| ✅ 화상/출혈 간접 지원 | 특수 심볼(불씨/가시) 추가 등장, 기존 화상/출혈 지속턴 연장 등 (직접 부여는 불가하지만 간접 강화는 가능) |
| ✅ 두두 상태이상 해제 | 화상/출혈/어지러움/빙결을 특화 해제 (아래 §3-1 참고) |

### 3-1. 두두 상태이상 해제 배분 원칙

> 두두에게 걸리는 상태이상 4종(화상/출혈/어지러움/빙결)의 해제는 아래 규칙을 따른다.

**두두 상태이상 4종:**

| 상태이상 | 효과 | 위협도 |
|---|---|---|
| 화상 | 심볼 2개 발화. 체크 시 최대체력 5% 피해 | ★★☆ |
| 출혈 | 스핀 버튼에 가시. 스핀 시 최대체력 7% 피해 | ★★★ |
| 어지러움 | 스핀 버튼 흔들림. 50% 확률 스핀 실패 | ★★★★ |
| 빙결 | 심볼 2개 동결. 스핀 불가 | ★★★☆ |

**해제 설계 규칙:**

1. **A등급 이상에서만 등장** — C~B 구간은 상태이상의 위협을 온전히 경험하는 구간
2. **패시브 위치** — 해제는 액티브가 아닌 패시브에 배치. 보조적 느낌 + "강화해서 얻었다" 감각
3. **강화 해금** — 최초 획득 시 없고, 강화를 통해 해금되는 슬롯에 배치
4. **특화 해제 우선** — 한 펫이 여러 상태이상을 해제하지 않음 (전용 해제 펫 제외)
5. **테마 연결 필수** — 해제와 펫 컨셉의 테마적 연결이 있어야 함

**현재 배분:**

| 등급 | 펫 | 해제 대상 | 테마 근거 |
|---|---|---|---|
| A | 설원 임프 | 화상 | 얼음이 불을 끔 |
| A | 타락 정령 | 어지러움 | 어지러움의 달인 = 해독도 앎 |
| A | 꽃의 정령 | 출혈 | 꽃잎이 상처를 봉인 |
| S | 블루혼 | 화상 | 얼음 계열 → 설원 임프 계보 |
| S | 가시 망이 | 어지러움 | 어지러움 계열 → 타락 정령 계보 |
| S | 눈릴라 | 빙결 | 얼음 요새 = 빙결 면역 |
| S | 마녀의 조형물 | 전체 해제 | 정화의 그릇 (전용 해제 펫, 출혈 포함) |
| SS | 깨달은 자 | 전체 해제 + 면역 | 깨달음의 경지 (전용 해제 펫) |

## 4. 펫 성장 시스템과 스킬 구조

### 4-1. 등급별 스킬 슬롯

| 등급 | 수량 | 액티브 | 최초 패시브 | 최대 패시브 | 강화 횟수 |
|---|---|---|---|---|---|
| C | 4종 | 1개 | 0개 | 0개 | 0회 |
| B | 8종 | 1개 | 0개 | 1개 | 1회 |
| A | 8종 | 1개 | 0개 | 2개 | 2회 |
| S | 8종 | 1개 | **1개** | 3개 | 2회 |
| SS | 8종 | 1개 | **2개** | 4개 | 2회 |

### 4-2. 패시브 해금 순서 = 설계 우선순위

```
B등급:  [강화1] 패시브1
A등급:  [강화1] 패시브1 → [강화2] 패시브2
S등급:  [최초] 패시브1 → [강화1] 패시브2 → [강화2] 패시브3
SS등급: [최초] 패시브1+2 → [강화1] 패시브3 → [강화2] 패시브4
```

**핵심 원칙:**
- **먼저 해금되는 패시브일수록 액티브와의 핵심 루프를 담당해야 함**
- S등급 패시브1 = 액티브 없이는 의미가 없지만, 있으면 핵심 루프가 완성되는 효과
- SS등급 패시브1+2 = 이 두 개만으로 SS의 판타지가 작동해야 함
- 뒤쪽 패시브(강화 해금) = 루프의 효율을 높이거나, 부가 혜택을 제공

### 4-3. 획득 경로

- **뽑기**: S등급까지 획득 가능
- **합성**: 동일 등급 최대강화 × N마리 → 랜덤 상위 등급 (S까지)
- **SS 제작**: 지정된 S++ 2마리 → 특정 SS 제작 (8종의 S → 8종의 SS, 각 S는 재료로 2회 사용)

## 5. 패시브 설계 원칙

### 5-1. 절대 원칙: 패시브는 "발동형"이어야 한다 — "상시형 수치 버프"는 장비의 영역

> **발동형 패시브 (허용)**: 펫이 특정 시점에 개입하여 효과를 부여하는 느낌.
> **상시형 패시브 (금지)**: 조건만 맞으면 자동으로 수치가 올라가는 느낌 — 펫이 보이지 않음.

- ✅ 발동형: "전투 시작 시 2턴간 랜덤 세로 1줄에 +2 부적" → 펫이 시작 시점에 부적을 뿌려줌
- ✅ 발동형: "조커 심볼이 체크되면 [액티브] 쿨타임 1 감소" → 펫이 반응하여 쿨타임을 깎아줌
- ✅ 발동형: "방패 소모 시 적에게 저체온 1스택" → 방패가 깨지는 순간 펫이 행동
- ❌ 상시형: "7콤보 달성 시 해당 턴 크리 데미지 +20%" → 펫과 무관한 조건부 수치 증가
- ❌ 상시형: "항상 두두 공격력 5% 증가" → 장비 패시브
- ❌ 상시형: "화상이 걸린 적에게 데미지 +15%" → 펫의 행동 없이 자동 적용

**판별 질문**: "이 효과를 발동시키는 주체가 펫인가?" → Yes면 발동형, No면 상시형.

### 5-2. 패시브 트리거는 즉시 인식 가능해야 한다

> "[액티브] 발동 중"처럼 발동 시점이 애매한 표현을 사용하지 않는다.
> 패시브의 조건은 유저가 **"지금 이게 발동했다"**를 바로 알 수 있어야 한다.

- ✅ 좋은 예: "조커 심볼이 체크되면" → 조커가 체크됨 = 즉시 인식 가능
- ✅ 좋은 예: "적 공격이 빗나가면" → 빗나감 = 즉시 인식 가능
- ✅ 좋은 예: "7콤보 달성 시" → 콤보 카운터로 즉시 인식 가능
- ❌ 나쁜 예: "[액티브] 발동 중 5콤보 달성 시" → "발동 중"의 범위가 모호
- ❌ 나쁜 예: "두두에게 공격력 버프가 있을 때" → 어떤 버프인지 구분 불가

### 5-3. 패시브의 4가지 유형 (허용)

1. **쿨타임 순환 (B1계열)**: 액티브를 더 자주 쓸 수 있게 해줌
2. **액티브 강화 (B2계열)**: 액티브의 효과를 더 좋게 해줌
3. **펫 고유 (B3계열)**: 펫만 가능한 특수 효과 (펫 간 시너지 등, SS 전용)
4. **펫 독자 행동 (B4계열)**: 펫이 특정 주기/조건에서 스스로 효과를 발동함

**B4계열 규칙:**
- 형태: "매 N라운드 시작 시 ~" / "전투 시작 시 ~" 등 고정 타이밍 발동
- 발동 주기: **최소 3라운드 이상** (매 턴 발동은 상시형과 구분 불가 → 금지)
- 등장 등급: **A등급 이상** (B등급에서는 쿨타임 순환/액티브 강화에 집중)
- 펫의 존재감: 유저가 "아, 이 펫이 해줬구나"를 느낄 수 있어야 함
- 예시:
  - ✅ "매 4라운드 시작 시 두두를 최대체력 10% 회복" → 주기 명확, 펫이 주체
  - ✅ "매 3라운드 시작 시 적에게 저체온 1스택 부여" → 주기 명확, 펫이 공격
  - ✅ "전투 시작 시 2턴간 랜덤 가로 1줄에 +2 부적" → 이미 꽃담비(S) P1에서 사용 중
  - ❌ "매 턴 두두 공격력 3% 증가" → 매 턴 = 상시형과 동일, 금지

### 5-4. 펫 특성: "언제 강한 펫인가" (A등급 이상 필수)

> A등급 이상 모든 펫은 **"이 펫은 언제 강한가?"**에 대한 답이 있어야 한다.
> 특성은 패시브에 녹여서 표현하며, 별도 시스템이 아니다.

| 특성 | 의미 | 설계 패턴 |
|---|---|---|
| 초반폭발 | 1~3턴 강함, 이후 약해짐 | 전투 시작시 쿨감, 프론트로드 효과 |
| 후반특화 | 후반에 폭발 | 누적 스택, 라운드 수 연동 |
| 위기특화 | 체력 낮을수록 강함 | HP% 조건 트리거 |
| 주기발동 | N라운드마다 피크 | 발동형(B4) 패시브 중심 |
| CC연동 | 적 상태이상 시 강함 | CC 조건 트리거 |
| 콤보연동 | 하이콤보 시 강함 | 콤보 임계치 조건 |

**편성 판단에 활용:**
- "이 보스 아프니까 위기형 데려가자" / "보스전 길어지니까 후반형으로"
- 초반형 + 후반형 = 전 구간 커버 / 위기형 + 공격형 = 상황별 전환
- 각 등급에서 6가지 특성이 모두 최소 1종 이상 존재해야 함

## 6. 등급별 차별화 원칙

### 6-1. 핵심: 새 등급 = 새로운 경험

> 단순히 숫자가 커지거나 스케일이 올라가는 것은 가급적 피한다.
> 단, 이 규칙을 너무 딱딱하게 적용하면 스킬 만들 때 제약이 커지므로 부분적으론 허용한다.

| 등급 전환 | 차별화 방법 | 허용 범위 |
|---|---|---|
| C → B | 패시브 1개 추가로 "시너지 루프" 첫 경험. 상태이상(저체온/어지러움) 첫 등장 | 액티브는 수치 스케일업 허용 |
| B → A | 새로운 심볼/역할 등장 (조커, 반반, 속성 특화) | 기존 카테고리 내 수치 증가도 부분 허용 |
| A → S | 복합 효과 액티브. 빌드 정체성 확립. **패시브도 A에 없던 새 메커니즘 필수** | 단순 스케일업 지양. 액티브와 패시브 모두에서 새로운 기능 필수 |
| S → SS | **전설급. 전투 규칙이 바뀌는 느낌** | 수치 스케일업만으로는 절대 불가 |

### 6-1-1. C등급 제한: 상태이상/슬롯 조작 금지

> C등급은 순식간에 지나가는 펫임. C에서 도입한 개념은 B에서 "새로운 경험"이 될 수 없음.
> C등급은 **공격력 증가, 보호막, 회복** 등 가장 기본적인 RPG 감각만 다룬다.
> 저체온/어지러움/부적/조커 등은 **B등급 이상**에서 처음 등장해야 함.

### 6-1-2. A→S 차별화 체크리스트

> A→S 구간에서 수치만 증가하고 패시브가 동일한 것은 명백한 실패.
> 아래 중 최소 2가지 이상 충족해야 함:
> - [ ] 액티브의 메커니즘이 A에서 불가능한 복합 효과인가?
> - [ ] 패시브 중 A에 존재하지 않는 새로운 트리거/효과가 있는가?
> - [ ] A와 같은 카테고리더라도, 전투 중 "다르게 운용"해야 하는 차이가 있는가?

### 6-2. 펫 컨셉 일관성 규칙

> **펫의 컨셉은 액티브 → 패시브 전체에 걸쳐 일관되어야 한다.**
> 액티브의 테마/판타지가 모든 패시브에서도 자연스럽게 느껴져야 함.

- ✅ 좋은 예: 방패 펫 → 패시브 "방패 소모 시 저체온" (방패가 깨지면서 얼음 파편이 튄다 = 자연스러움)
- ❌ 나쁜 예: 거미줄 펫 → 패시브 "저체온 부여" (거미줄과 저체온은 테마적 연결 없음)
- 펫 이름/컨셉과 모든 스킬 효과가 **한 문장으로 연결 가능해야** 함
  - "이 펫은 OO하는 펫이다" → 모든 액티브/패시브가 이 문장에서 벗어나지 않음

### 6-2-1. S등급 이상 듀얼 컨셉 필수

> S등급 이상은 단일 컨셉으로 패시브 3~4개를 채우기 어렵고, 단일 컨셉만으로는 밋밋함.
> 두 가지 역할을 조합하여 펫의 전략적 깊이를 만든다.

- **S등급**: 최소 2개 컨셉 조합 (예: 조커+빙결, 어지러움+방패, 분노+크리)
- **SS등급**: 2개 컨셉 + 고유 메커니즘 (예: 연쇄스핀+부적, 거미줄+CC)
- 두 컨셉 간 시너지가 있어야 함 (A가 B를 강화하거나, A의 결과가 B의 조건이 되는 구조)

### 6-3. SS등급 특별 규칙

- **글을 읽는 것만으로 갖고 싶어지는 매력이 있어야 함**
- RPG의 "전설급 스킬" 느낌
- S등급에 없는 **완전히 새로운 메커니즘** 최소 1개 포함
- 필요 시 SS 펫만의 **신규 개념** 도입 가능

### 6-4. SS 채택 신규 개념 (4종)

| 개념 | 정의 |
|---|---|
| 연쇄 스핀 | 추억장식 발동 전에 리스핀 1회 추가. 기존 매칭 위에 추가 매칭 축적 |
| 가로 이중 체크 | N턴간 모든 가로 체크가 2회 카운트됨. 데미지·콤보·효과 전부 포함 |
| 펫 전용 특수 심볼 | SS 펫 고유 심볼이 그리드에 등장. 체크 시 고유 효과 (1~2종만 적용) |
| DoT 추가 발동 | N턴간 매 스핀마다 화상/출혈 DoT가 1회 추가로 발동됨 |

### 6-5. 기각된 개념

| 개념 | 기각 사유 |
|---|---|
| 적 패시브 봉인 | 적의 아이덴티티를 완전히 죽임. 너무 만능이라 전략적이지 않음 |
| 추억장식 증폭 | 추억장식 곱배율 자체가 이미 강력한 뻥튀기 요소. 2배 확률 발동은 밸런스 붕괴 위험 |
| 심볼 홀드 | "이전 스핀 체크된 심볼 고정 + 나머지만 리스핀" — 리프 버드의 연쇄 스핀과 느낌이 너무 유사. 대나무 담비 설계 단계에서 기각됨 |

## 7. 쿨타임 설계 기준

### 7-1. 전투 길이

| 상황 | 턴 수 |
|---|---|
| 일반 몹 | 6~7턴 |
| 보스 | ~15턴 |

### 7-2. 쿨타임 가이드

| 등급 | 쿨타임 범위 | 일반몹 발동 횟수 | 보스 발동 횟수 |
|---|---|---|---|
| C | 8~9 | 0~1회 | 1~2회 |
| B | 7~8 | 1회 | 2회 |
| A | 7~8 | 1회 | 2회 |
| S | 6~7 | 1회 | 2~3회 |
| SS | 6~7 | 1회 | 2~3회 |

- 일반몹전에서 액티브가 최소 1회는 발동해야 펫의 존재감이 있음
- 패시브의 쿨타임 감소가 2회 발동을 가능하게 하는 것이 이상적
- 보스전에서는 쿨타임 순환 루프가 빛나야 함

## 8. 인게임 스킬과의 시너지 원칙

### 8-1. 펫은 인게임 스킬 빌드를 "완성"시키는 역할

- 인게임 스킬 = 로그라이크 드래프트로 매 런마다 다른 빌드
- 펫 = 특정 빌드 방향을 강화하는 고정 장비

**예시:**
- 빙결 빌드(얼음+영구동토) + 저체온 펫 → 빙결 빈도 대폭 상승
- 패턴 빌드(가로 강화+삼각형) + 부적 펫 → 가로 매칭 보장
- 크리 빌드(크리확률+크리피해) + 크리 버프 펫 → 폭발 데미지

### 8-2. 펫끼리의 역할 분담

2마리 편성에서 **"무엇을 포기하느냐"**가 전략적 고민이어야 함.

- 공격 + 공격 = 딜 극대화, 생존 불안
- 공격 + 보호 = 균형형
- 디버프 + 디버프 = CC 체인
- 슬롯 + 버프 = 매칭 + 데미지 극대화

**정답 조합이 하나가 아니라, 인게임 스킬 빌드에 따라 최적 조합이 달라져야 함.**

---

## 8-3. 등급 포괄 원칙

> **상위 등급은 하위 등급의 역할 개념을 포괄해야 한다.**

A등급 펫을 합성 소모하여 S등급을 만드는 구조에서, A에 존재하던 역할군이 S에 없으면 **"승급했는데 할 수 있는 게 줄었다"** = 성장 역행이 발생함. 이는 치명적 설계 결함임.

**규칙:**
- A에 존재하는 역할군(부적, 크리, 저체온, 어지러움, 해제 등)은 S에도 같은 역할군의 펫이 최소 1종 이상 존재해야 함
- 물리적으로 동일한 스킬일 필요는 없으나, 같은 전략적 목적을 달성할 수 있어야 함
- 검증: 등급 전환 시 "이 역할을 할 수 있는 펫이 상위에도 있는가?" 체크 필수
- 이 원칙은 B→A, A→S, S→SS 모든 등급 전환에 적용됨

---

## 9. SS 제작 조합 설계 원칙

- S 8종 → SS 8종, 각 S는 재료로 정확히 **2회** 사용
- 조합의 테마적 연결이 있어야 함 (두 S의 역할이 SS에 녹아있어야 함)
- 유저가 "이 SS를 만들려면 이 두 S를 먼저 키워야 하는구나"라는 목표가 생겨야 함

---

## 10. 스킬 텍스트 작성 규칙

- 효과를 **한 문장**으로 설명할 수 있어야 함
- 조건부 효과는 최대 **2개**까지 (3개 이상이면 이해 불가)
- 구체적 수치 명시 (애매한 표현 금지: "조금 증가" ❌ → "15% 증가" ✅)
- SS등급이라도 패시브 1개당 설명이 **2줄을 넘지 않아야** 함

## 11. 신규 로직/메커니즘 도입 시 검증 규칙

> 기존에 없는 새로운 메커니즘을 도입할 때는, 반드시 아래 항목을 모두 검증한 후에 채택한다.
> 하나라도 답변이 불명확하면 도입하지 않는다.

```
□ 발동 타이밍이 명확한가? (체크 전? 후? 데미지 적용 전? 후?)
□ 대상 선택이 명확한가? (랜덤? 특정 조건? 유저 선택?)
□ 무한 루프/무한 반복이 발생할 수 없는가?
□ 유저가 처음 읽었을 때 한 번에 이해할 수 있는가?
□ 구현 난이도가 합리적인가? (개발팀이 감당 가능한 수준인가?)
□ 기존 메커니즘과 충돌하거나 예외 케이스를 만들지 않는가?
```

이 검증을 통과하지 못한 메커니즘은 **아무리 재미있어 보여도 기각**한다.
