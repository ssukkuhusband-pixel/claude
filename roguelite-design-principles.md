# 로그라이트 설계 원칙 바이블

> 모바일 15종 + PC/콘솔 15종 = 총 30종 로그라이트 게임 데이터에서 귀납적으로 도출한 설계 원칙
> 구체적 수치/사례가 필요하면 `research/roguelite-systems/` 폴더의 원시 데이터를 참조한다

## 원칙 0. 로그라이트의 본질 — "랜덤 기계를 설계하는 재미"

> **유저가 확률의 결과를 "구경"하는 게 아니라, 확률 기계 자체를 "설계"하는 구조여야 한다.**

30종 분석에서 가장 중요한 단일 인사이트:

| 게임 | 유저가 하는 일 | 본질 |
|------|-------------|------|
| Balatro | 조커를 사서 족보 강화 기계를 조립 | 포커 기계 설계 |
| Luck be a Landlord | 심볼을 추가해서 슬롯 기계를 조립 | 슬롯 기계 설계 |
| Slay the Spire | 카드를 골라서 덱 엔진을 구축 | 카드 엔진 설계 |
| Hades | Boon을 선택해서 전투 스타일을 조립 | 전투 빌드 설계 |
| Vampire Survivors | 무기 조합으로 진화 트리를 완성 | 진화 기계 설계 |

**실패하는 로그라이트의 공통점**: 유저가 랜덤 결과를 수동적으로 받아들이기만 하는 구조. "좋은 거 나오길 기도" = 게임이 아니라 복권.

**적용 기준**: 비전통 메커닉(슬롯/포커/주사위 등) 로그라이트라면, 유저가 "확률 기계의 결과를 보는" 수동 경험이 아니라 "확률 기계 자체를 조립하는" 능동 경험으로 설계되어야 한다.

## 원칙 1. 런 시간 — 모바일은 5~15분이 절대 법칙

### 데이터

| 런 시간 | 대표 게임 | 수익 규모 |
|--------|----------|----------|
| 5~15분 | Archero, Capybara Go, 탕탕특공대 | $100M~$263M |
| 15분 고정 | Survivor.io | **$500M+** |
| 20~40분 | Soul Knight, Otherworld Legends | $수백만~수천만 |
| 30~60분 | Dead Cells, Hades(PC) | PC 프리미엄 영역 |

### 원칙

- **모바일: 한 판 5~15분.** 이 범위를 넘기면 이동 중 플레이 불가 + 중단 불가 → 이탈
- **PC: 한 판 20~60분.** 몰입 시간이 더 길지만, 90분 초과는 초반 trivial 구간 문제 발생
- 런이 짧을수록 "한 판만 더" 발동 빈도 증가 — Survivor.io가 15분 고정으로 $500M+ 달성한 핵심 이유
- 중단 가능 구조(자동 저장/일시정지) 필수 — 모바일은 언제든 전화/알림으로 중단됨

### 체크리스트

```
□ 한 판(런) 시간이 목표 범위 내인가? (모바일 5~15분 / PC 20~60분)
□ 중단 후 복귀 시 자연스럽게 이어지는가?
□ 런 시간 대비 의미 있는 선택 횟수가 충분한가? (최소 5회 이상)
```

## 원칙 2. 콘텐츠 풀 규모 — "다 알기 전에 질리지 않는" 균형

### 데이터

| 게임 | 총 콘텐츠 수 | 런 내 획득 수 | 획득/전체 비율 |
|------|------------|------------|-------------|
| Balatro | 234종 | 조커 5개 | 2.1% |
| Luck be a Landlord | 379종 | 심볼 15~25개 | 5~7% |
| Slay the Spire | 1,030+종(카드+렐릭) | 20~42개 | 2~4% |
| Binding of Isaac | 719종 | 15~30개 | 2~4% |
| Archero | 100+종 | 7~15개 | 7~15% |
| Survivor.io | 80+종 | 10~20개 | 12~25% |

### 원칙

- **총 콘텐츠: 100~400종이 스위트 스폿.** 100 미만은 빌드 다양성 부족, 400 초과는 희석 문제
- **런 내 획득 비율: 총 콘텐츠의 3~15%.** 이 비율이 "희소성 = 선택의 의미"를 만듦
- 모바일은 총 콘텐츠가 적어도 됨(80~150종) — 런이 짧으므로 런당 접하는 비율이 자연히 높아짐
- PC는 총 콘텐츠가 많아야 함(200~1,000종) — 런이 길어 같은 조합을 자주 봄

### 희석 방지 필수 장치

콘텐츠가 많아질수록 "원하는 아이템을 영원히 못 보는" 희석 문제 발생. 반드시 아래 중 하나 이상 적용:

1. **가중치 드롭 테이블** — 현재 빌드와 연관된 아이템 출현 확률 증가
2. **빌드 바이어스(Procedural Archetype)** — 유저의 빌드 방향을 감지해서 관련 아이템 가중치 자동 조정
3. **리롤(재추첨)** — 마음에 안 드는 선택지를 유료/제한 횟수로 교체
4. **클래스/캐릭터 분리** — Slay the Spire처럼 캐릭터별 카드 풀 분리 → 풀 크기 자체를 줄임
5. **상점 시스템** — 원하는 아이템을 인게임 화폐로 직접 구매 가능

### 체크리스트

```
□ 총 콘텐츠(스킬/아이템/심볼) 수는? 100~400종 범위인가?
□ 런당 획득 비율은 3~15%인가?
□ 희석 방지 장치가 최소 1개 이상 설계되어 있는가?
□ 콘텐츠 추가 시 희석 문제를 함께 검토하는가?
```

## 원칙 3. 등급(레어리티) — 3~5단계, 등급 차이는 메커니즘 변화

### 데이터

| 등급 수 | 해당 게임 수 (30종 중) | 비고 |
|--------|---------------------|------|
| 등급 없음 | 6종 (Vampire Survivors, Noita, Dicey 등) | 퓨어 로그라이크 계열 |
| 3단계 | 8종 (StS, Dead Cells, Dicey 등) | PC 인디 주류 |
| 4단계 | 7종 (Brotato, LbaL, Peglin 등) | PC/모바일 표준 |
| 5단계 | 5종 (Balatro, Hades, Archero 등) | 모바일 F2P 또는 대형 PC |
| 6단계+ | 0종 | 혼란 발생, 채택 사례 없음 |

### 원칙

- **3~4단계가 표준.** BM 목적이 있는 모바일 F2P는 5단계까지 허용
- **6단계 이상은 금지.** 30종 중 채택 사례 0. 유저 인지 과부하
- **등급 간 차이는 수치 증가가 아니라 메커니즘 변화여야 한다**
  - 나쁜 예: "화염 데미지 10% → 20% → 30%"
  - 좋은 예: "화염 데미지" → "화염 데미지 + 화상 DoT" → "화염 데미지 + 화상 DoT + 폭발 범위"
- 높은 등급일수록 **조건부 효과**가 붙어야 함 — 무조건 강한 게 아니라 "상황에 따라 강한" 설계

### 체크리스트

```
□ 등급 체계가 3~5단계 범위인가?
□ 등급 간 차이가 단순 수치가 아닌 메커니즘 변화를 포함하는가?
□ 높은 등급이 "항상 최선"이 아닌 "상황에 따라 최선"인가?
```

## 원칙 4. 스킬 선택 방식 — "3개 중 1택"이 모바일 황금률

### 데이터

| 선택 방식 | 채택 게임 | 모바일 수익 상관 |
|----------|---------|---------------|
| 레벨업 시 3택 | Archero, Survivor.io, Capybara Go, Hades | **최고 수익 게임 전부** |
| 쇼핑+3택 혼합 | Brotato, Soul Knight, Balatro | 중간~높음 |
| 드롭+즉시 결정 | Dead Cells, Tallowmere | PC 프리미엄 |
| 덱빌딩 카드 | StS, Meteorfall, Rogue Adventure | 소규모 수익 |

### 원칙

- **모바일 표준: 레벨업 시 3개 중 1택.** 인지 부담 최소, 선택의 쾌감 최대
- 2개 중 1택은 선택 느낌이 약함. 4개 이상은 모바일에서 비교 시간이 너무 김
- 선택지는 **즉시 효과가 체감되어야 함** — 선택 후 다음 전투에서 바로 변화를 느껴야 함
- 상점/쇼핑 시스템은 보조 선택 채널로 유효 — 경제 레이어 추가

### 체크리스트

```
□ 스킬 선택이 "3개 중 1택" 구조인가?
□ 선택 후 즉시 효과가 체감되는가?
□ 선택 빈도는 적절한가? (런 5~15분에 최소 5~10회 선택 기회)
```

## 원칙 5. 시너지 설계 — 개별은 평범, 조합이 강력

### 데이터

| 게임 | 시너지 구조 | 결과 |
|------|-----------|------|
| Vampire Survivors | 무기+패시브 → 진화(Evolution) | 조합 탐구 = 핵심 재미, 커뮤니티 자발 공유 → 바이럴 |
| Hades | 두 신의 축복 → Duo Boon 발동 | 발견의 재미, 전략적 신 선택 |
| Balatro | 조커 + 족보 → 칩×멀티 폭발 | 지수함수적 스케일링 = 감탄 |
| LbaL | 심볼 인접 상호작용 | 심볼 배치 최적화 탐구 |
| Survivor.io | 무기+무기 → EVO 진화 | 진화 루트 발견 동기 |

### 원칙

1. **개별 아이템은 평범하게, 조합이 강력하게** — 단독 최강 아이템은 시너지 탐구를 죽인다
2. **핵심 시너지 5~7개 + 보조 시너지 다수** — 핵심이 너무 적으면 빌드 고정, 너무 많으면 파악 불가
3. **일부 암시, 일부 발견** — 전부 공개하면 발견 재미 없음, 전부 숨기면 답답
4. **시너지 결과는 지수함수적 스케일링** — 선형 보너스(+10%)는 감탄을 못 만듦. 곱셈(×2, ×10)이 감탄을 만듦
5. **시너지가 "공유하고 싶은 경험"이 되어야 함** — 스크린샷/영상으로 자랑할 수 있는 구조 = 바이럴

### 시너지 계층 예시 (비전통 메커닉 로그라이트)

- 단일 요소 강화 = 기본 시너지 (암시 가능)
- 2~3요소 조합 = 상위 시너지 (발견 재미)
- 4요소+ 극한 조합 = 극한 시너지 (공유하고 싶은 경험)

### 체크리스트

```
□ 단독 최강 아이템/스킬이 존재하지 않는가?
□ 핵심 시너지 루트가 5~7개 설계되어 있는가?
□ 시너지 결과가 지수함수적 스케일링을 보이는가?
□ 시너지 일부는 유저가 스스로 발견하는 구조인가?
□ 조합 결과가 스크린샷/영상으로 공유하고 싶은 수준인가?
```

## 원칙 6. 함정 선택지(Trap Choice) — 삭제가 아니라 설계 도구

### Slay the Spire GDC 2019 핵심

> "Every card should have its place." — 모든 카드는 자기 자리가 있어야 한다.

### 원칙

- **함정 선택지는 숙련자와 초보자를 구분하는 장치** — 제거 대상이 아님
- 초보: "좋아보이는 것"을 선택 → 나중에 후회 → 학습
- 숙련자: "현재 빌드에 맞는 것"을 선택 → 최적화의 쾌감
- **완전한 함정(절대 쓸모없는 것)은 안 됨** — 반드시 "상황에 따라 좋은 선택"이 되어야 함
- **"항상 고르는 스킬"이 3개 이상이면 빌드 다양성 붕괴** — 메트릭으로 픽률 측정 → 조정 필수

### 체크리스트

```
□ 모든 스킬/아이템이 "자기 자리"가 있는가? (완전한 쓰레기가 없는가?)
□ "항상 고르는" 필수 픽이 3개 이상 존재하지 않는가?
□ 숙련도에 따라 선택 패턴이 달라지는 구조인가?
□ 픽률 메트릭을 측정할 계획이 있는가?
```

## 원칙 7. 난이도 곡선 — "3의 법칙"과 파워 커브

### "3의 법칙" (30종 공통 패턴)

```
3 스테이지마다 → 강화 1회 (스킬 선택, 아이템 획득)
5 스테이지마다 → 엘리트/중간 보스 (예열)
10 스테이지마다 → 챕터 보스 (클라이맥스)
```

### 파워 커브

```
[시작] 약한 상태 → "성장해야 한다" 동기 부여
       ↓
[초반] 빠른 강화 → 성장감 즉시 체감
       ↓
[중반] 가속 → 빌드 방향 확정, 시너지 발동 시작
       ↓
[후반] 정점 → 빌드 완성, 최대 파워 체감
       ↓
[보스] 정점 vs 최강 적 → 클라이맥스
```

### 주의사항

- **"죽음의 계곡"**: 약한 상태가 너무 오래 지속 → 이탈. 초반 3 스테이지 안에 첫 강화 필수
- **초반 trivial화**: 영구 성장이 쌓이면 초반이 "지루한 의무"가 됨 → Dead Cells, Archero에서 비판받은 문제
  - 해결: 영구 성장이 초반 스킵 기능을 제공하거나, 초반 난이도를 동적으로 조정

### 체크리스트

```
□ 첫 강화가 3 스테이지 안에 발생하는가?
□ 보스 직전에 유저 파워가 정점에 도달하는가?
□ 영구 성장 후에도 초반이 지루하지 않은가?
□ 엘리트가 보스 전 "예열" 역할을 하고 있는가?
```

## 원칙 8. 영구 성장(메타 프로그레션) — 콘텐츠 언락 > 스탯 성장

### 30종 비교

| 메타 강도 | 게임 | 결과 |
|----------|------|------|
| 없음 | Brotato, Noita | 퓨어 로그라이크, PC 하드코어 전용 |
| 약함 (콘텐츠 언락) | StS, Dead Cells | 건강한 리플레이, PC 평가 최상 |
| 중간 (스탯+콘텐츠) | Hades, Vampire Survivors | 접근성+깊이 양립, 가장 넓은 유저층 |
| 강함 (직접 스탯) | Rogue Legacy, Archero | 진입장벽 낮음, 그라인드 의존 위험 |
| 매우 강함 | Archero, Survivor.io | 모바일 BM 연결, 로그라이트 정체성 위험 |

### 원칙

- **"절대 헛된 런이 없다"** (Hades 철학) — 매 런에 최소 1개의 영구 보상
- **콘텐츠 언락 > 스탯 성장** — 스탯 성장은 "충분히 그라인드하면 클리어" = 스킬 기반 재미 사망
- **런 내 선택이 결과를 80% 결정, 메타 성장은 20%** — 이 비율이 깨지면 로그라이트가 아니라 RPG 그라인드
- 모바일에서 메타 성장이 더 강한 것은 현실이지만, **스탯 아닌 옵션 확장으로 가는 게 이상적**

### 모바일에서 메타가 필수인 4가지 이유

1. 리텐션: "모든 게 사라지면" 모바일 유저 이탈
2. BM 연결: 성장 재화 가속 구매
3. 세션 길이: 짧은 세션에서 완전 리셋은 가혹
4. 캐주얼 배려: 모바일 하드코어 비율 < PC

### "죽어서 배운다" vs "죽어서 강해진다" vs "죽어서 이야기가 진행된다"

| 철학 | 대표 | 리텐션 유형 |
|------|------|-----------|
| 죽어서 배운다 | StS, Noita | 마스터리 지향 유저 |
| 죽어서 강해진다 | Rogue Legacy, Archero | 캐주얼 포함 넓은 층 |
| 죽어서 이야기가 진행된다 | **Hades** | **가장 넓은 유저층** |

### 체크리스트

```
□ 매 런에 영구 보상이 최소 1개 있는가?
□ 영구 성장이 "스탯 증가"가 아닌 "옵션 확장"에 가까운가?
□ 런 내 선택이 결과의 80%를 결정하는가? (메타 성장 ≤ 20%)

```

## 원칙 9. 리플레이 동기 — 4가지 심리 엔진

### "한 판만 더"의 4가지 원천

1. **클리어 직전 절박감** — "보스 HP 10%까지 깎았는데!" → 즉시 재시도
2. **미완성 언락** — "이번 런에서 새 캐릭터 언락 직전까지 갔는데" → 다음 런 동기
3. **빌드 호기심** — "화염+무지개 조합 해보고 싶다" → 아이디어 큐 형성
4. **내러티브 진행** — Hades: 죽어도 NPC 대화 진행 → 스토리 보러 다시 플레이

### 언락 시스템 설계

| 방식 | 장점 | 단점 | 추천 |
|------|------|------|------|
| 콘텐츠 언락 | 건강한 성장, 게임이 커짐 | 초반 빌드 제한 | PC 로그라이트 |
| 파워 언락 | 진입장벽 낮음, 좌절 완화 | 그라인드 의존 | 모바일 캐주얼 |
| 혼합 (Hades식) | 균형, 넓은 유저층 | 설계 복잡도 높음 | **최적 선택** |

### 체크리스트

```
□ 4가지 리플레이 동기 중 최소 2개가 설계에 존재하는가?
□ 세션 종료 시 유저에게 "미완의 목표"가 최소 1개 남는가?
□ 언락 시스템이 "파워"가 아닌 "콘텐츠/옵션 확장" 위주인가?
```

## 원칙 10. 스케일링 — 지수함수가 감탄을 만든다

### 데이터

| 게임 | 스케일링 구조 | 결과 |
|------|------------|------|
| Balatro | 칩(C) × 멀티(M) = 지수함수 | ×1 → ×10,000 도달, 유저 감탄 |
| LbaL | 덧셈 심볼 → 곱셈 심볼 추가 | 후반 수익 폭발 |
| Vampire Survivors | 무기 진화 → 화면 가득 채움 | 시각적 쾌감 폭발 |
| Archero | 선형 스탯 + 스킬 중첩 | 보통 수준 |
| Dicey Dungeons | 상대적 선형 | 감탄 약함 |

### 원칙

- **선형 성장(+10%, +20%)은 감탄을 만들지 못함** — "좀 강해졌네" 수준
- **지수 성장(×2, ×10, ×100)은 감탄을 만듦** — "미쳤다!" 수준
- **스케일링 구조 = 공유하고 싶은 경험** — 화면 가득 데미지/이펙트 = 스크린샷 → 바이럴
- Balatro의 성공 핵심: 포커 족보가 현실에서 불가능한 수준으로 과장 → "포커 판타지" 실현

### 스케일링 예시 (비전통 메커닉)

```
Balatro: 기본 족보 100칩 → 조커 1개 ×3 → 조커 3개 ×3×5×2 = 3,000칩 (지수)
LbaL: 기본 심볼 $1 → 곱셈 심볼 추가 → $1 × 5 × 3 = $15 (지수)
```
이런 지수 스케일링이 "스크린샷 찍고 싶은 순간"을 만들어야 한다.

### 체크리스트

```
□ 빌드 완성 시 데미지/효과가 지수함수적으로 폭발하는가?
□ "미쳤다!" 순간이 런당 최소 1~2회 발생하는가?
□ 스케일링 결과가 시각적으로 화려하게 표현되는가?
```

## 비전통 메커닉 로그라이트 — 4종 비교 분석

### 4종 핵심 비교

| 항목 | Balatro (포커) | LbaL (슬롯) | Peglin (파칭코) | Dicey Dungeons (주사위) |
|---|---|---|---|---|
| 판매량 | **500만+** | 50~100만 | 50~100만 | 10~30만(추정) |
| 런 시간 | 30~60분 | 20~40분 | 60~75분 | 20~40분 |
| 운/실력 비율 | 30/70 | 40/60 | 50/50 | 60/40 |
| 스케일링 | **지수함수(×10,000)** | 지수(곱셈 심볼) | 체인 증폭 | 선형 |
| 핵심 | 포커 판타지 | 슬롯 큐레이션 | 물리 엔진 전투 | 캐릭터별 규칙 변화 |

### 성공 규모 차이의 원인 분석

Balatro(500만)가 나머지(50만 이하)를 압도한 이유:
1. **보편적 메커닉**: 포커 = 전 세계가 아는 게임 → 진입장벽 최저
2. **지수 스케일링**: 칩×멀티 폭발 = "미쳤다!" 순간 빈발 → 바이럴
3. **비주얼 명확성**: 숫자가 커지는 것이 곧 쾌감 → 시각 피드백 최적

**비전통 메커닉 로그라이트 설계 시사점:**
- 보편적 메커닉(포커/슬롯/주사위 등 전 세계가 아는 것)일수록 진입장벽 낮음
- 스케일링이 지수함수적이어야 "미쳤다!" 순간 → 바이럴
- 비전통 메커닉의 결과가 시각적으로 명확해야 함 (숫자가 커지는 것 = 쾌감)
