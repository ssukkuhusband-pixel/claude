## 원칙 0. 소셜의 본질 — "코어 루프는 솔로, 결과는 공유"

> 소셜은 반드시 메타 레이어(보상/보스/랭킹)에 얹어야 한다.

## 원칙 2. 길드/클랜 — 최고의 리텐션 앵커

### 설계 기준

| 항목 | 기준 |
|------|------|
| 길드 인원 | **20~50명** (20 미만은 활동 부족, 50 초과는 유대감 약화) |
| 일일 기부 부담 | **1분 미만** (과하면 번아웃) |
| 참여 방식 | **선택적 보너스** (강제 금지) |
| 길드 보스 | **데일리 앵커** = 매일 로그인 이유 (AFK Arena, Capybara Go 검증) |
| 길드 채팅 | 기본 제공, 소속감의 핵심 |
| 길드 상점 | 기여도로 구매 → 과금 포인트 연결 |

### 길드 보스 = 로그라이트 최적 소셜 구조

```
[일일 루프]
로그인 → 길드 보스 도전 → 개인 런(솔로) → DPS 기록 → 길드 기여도 적립
                                                     ↓
                                              길드 보상 분배 → 개인 성장에 기여
```

- 코어 루프(솔로 런) 침범 없이 소셜 연결
- AFK Arena, Capybara Go에서 검증
- **운수좋은날 적용**: 길드 보스에게 슬롯 전투 → 길드원 전체 DPS 합산 → 보상

### 체크리스트

```
□ 길드 인원이 20~50명 범위인가?
□ 일일 기부/참여 부담이 1분 미만인가?
□ 길드 보스(또는 유사 데일리 앵커)가 있는가?
□ 참여가 강제가 아닌 보너스인가?
```

## 원칙 3. 비동기 PvP > 실시간 PvP (모바일)

### 5가지 구조적 이유

| # | 이유 | 데이터 |
|---|------|--------|
| 1 | 동시 접속 불필요 | 매칭 실패 = 이탈. 비동기는 매칭 문제 없음 |
| 2 | 세션 중단 가능 | 모바일 세션 3~8분, 전화 오면 즉시 끊김 |
| 3 | 분노-복귀 루프 | Coin Master: "공격당함" 알림 → 분노 복귀 → $6B |
| 4 | P2W 밸런스 용이 | AI 방어대 = 스펙 차이 → 성장 과금 자연 연결 |
| 5 | 인프라 비용 절감 | DB 조회만으로 PvP 구현 가능 |

### 비동기 PvP가 BM에 유리한 핵심 이유

> **"손실 복구"가 즉각 과금 포인트가 된다.**
> 실시간 PvP에서 진 것은 과금으로 회복 불가.
> 비동기 공격으로 잃은 자원은 과금으로 즉각 복구 가능.
> 이것이 Coin Master $6B의 진짜 비밀이다.

### 로그라이트 비동기 PvP 형태

| 형태 | 원리 | 적합도 |
|------|------|--------|
| 빌드 vs 빌드 (AI 조종) | 내 아웃게임 장비 빌드 vs 상대 빌드, AI 대전 | ★★★★★ |
| 런 결과 스코어 랭킹 | 주간 챌린지 동일 조건 → 점수 비교 | ★★★★★ |
| 길드 DPS 경쟁 | 각자 솔로 런 → 길드 합산 | ★★★★☆ |
| Ghost PvP | 상대의 과거 런 기록 vs 내 실시간 런 | ★★★★☆ |

### 운수좋은날 비동기 PvP 설계안

```
[슬롯 대전]
내 아웃게임 장비+펫+추억장식 → "방어 슬롯" 자동 구성
상대가 "공격 슬롯"으로 도전 → AI가 방어 슬롯 운영
결과: 승패에 따라 랭킹 포인트 + 보상

→ 코어 루프(인게임 슬롯 전투)와 동일한 메커닉
→ 아웃게임 장비 강화 = PvP 경쟁력 = 과금 동기
→ 컨셉 통일: 전투도 슬롯, PvP도 슬롯
```

### 체크리스트

```
□ PvP가 비동기 방식인가? (모바일 기준)
□ 매칭 대기 시간이 0에 가까운가?
□ 패배 시 "손실 복구" 과금 포인트가 존재하는가?
□ PvP가 코어 루프와 동일한 메커닉인가? (컨셉 통일)
```

## 원칙 5. 소셜 동기 4가지 — 리텐션과 BM은 순서가 다르다

### 소셜 동기 4가지

| 동기 | 리텐션 기여 순위 | BM 기여 순위 | 설명 |
|------|---------------|-------------|------|
| **소속감** | 1위 | 3위 | "내 팀이 있다" → 이탈 방지 |
| **경쟁** | 2위 | 2위 | "이기고 싶다" → 참여 + 과금 |
| **협력** | 3위 | 4위 | "같이 한다" → 일일 로그인 |
| **과시** | 4위 | **1위** | "보여주고 싶다" → 코스메틱 구매 |

### 원칙

- 리텐션을 원하면 **소속감**(길드)을 먼저 설계
- BM을 원하면 **과시**(코스메틱/랭킹)를 먼저 설계
- **이상적: 소속감 + 과시를 동시에** — 길드 내에서 코스메틱으로 과시하는 구조

### 체크리스트

```
□ 소속감(길드)이 설계되어 있는가?
□ 과시(코스메틱/랭킹 표시)가 설계되어 있는가?
□ 두 가지가 연결되어 있는가? (길드 내 과시)
```

## 원칙 6. 소셜-BM 연계 — 4가지 수익 경로

| 경로 | 원리 | 효과 |
|------|------|------|
| **길드 기부** | "팀을 위해" 결제 심리 | 재화 구매 직접 유도 |
| **PvP 시즌 + 배틀패스** | PvP 미션 = BP 조건 | 참여율+BP 구매 동시 상승 |
| **코스메틱 과시** | 팀 내/랭킹 내 과시 | 30시간+ 유저의 47%가 구매 |
| **협동 역할 다양성** | 다른 역할 필요 → 다른 캐릭/장비 | 가챠 수요 자연 창출 |

### 운수좋은날 적용

```
길드 기부 → 슬롯 전투 보상 일부를 길드에 기부 → 길드 상점 포인트
PvP 시즌 → 주간 슬롯 대전 랭킹 → 시즌패스 미션 연동
코스메틱 → 슬롯머신 스킨, 심볼 이펙트 → 길드/PvP에서 과시
협동 → 길드 보스에 속성별 역할 → 다양한 장비/펫 필요
```

### 체크리스트

```
□ 소셜 기능이 BM과 연결되는 경로가 최소 2개 있는가?
□ 코스메틱이 과시 가능한 맥락(길드/PvP)에서 보이는가?
□ PvP가 시즌패스/배틀패스와 연동되는가?
```

## 원칙 8. Coin Master 모델 — 슬롯+PvP의 검증된 공식

### Coin Master가 증명한 것 ($6B)

```
슬롯(랜덤) + 비동기 PvP(공격/약탈) = 모바일 최적 조합

왜 작동하는가:
1. 슬롯 랜덤성 → P2W 격차 심리 희석 ("나도 운 좋으면 이길 수 있다")
2. 비동기 공격 → "공격당함" 알림 → 분노 복귀 → 리텐션 엔진
3. 공격으로 잃은 자원 → 스핀(과금) 구매 → BM 직결
4. 빌리지 건설(메타게임) → 장기 목표 → 장기 리텐션
```

### 운수좋은날 직접 적용

운수좋은날은 슬롯 기반 게임이므로 Coin Master의 핵심 구조를 참고할 수 있다:

```
Coin Master         →  운수좋은날
스핀(슬롯)           →  슬롯 전투(인게임)
빌리지 건설          →  장비/펫 강화(아웃게임)
비동기 공격/약탈     →  비동기 슬롯 대전
공격당함 알림        →  "방어 실패" 알림 → 복귀
스핀 고갈 → 구매    →  에너지/스핀 고갈 → 구매
```

### 체크리스트

```
□ 슬롯의 랜덤성이 PvP 격차를 심리적으로 희석하는 구조인가?
□ 비동기 PvP 패배 시 "공격당함" 알림이 복귀를 유도하는가?
□ 패배 손실의 복구가 과금 포인트와 연결되는가?
□ 메타게임(장비 강화 등)이 PvP 참여 동기를 만드는가?
```
