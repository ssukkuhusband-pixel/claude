## 1. 소셜 시스템 설계 원칙

### 1-1. 길드/클랜 시스템의 핵심 설계 요소

**출처: GDC Europe 2013, "Building Games for the Long Term: Pragmatic F2P Guild Design" (Anthony Pecorella, Kongregate)**

길드 시스템의 핵심 구성요소:
- **허브 구조**: 길드는 독립 기능이 아닌 다른 소셜 기능들의 허브 역할
  - 기부/대여 시스템
  - 협동 플레이
  - 길드 전쟁/토너먼트
  - 공유 목표(Guild Quest)
- **노미네이션 메커닉**: 길드 리더/임원 지명 → 특정 유저를 슈퍼 인게이지 상태로 만듦 → 길드 이벤트 중 효율적 운영
- **길드 전쟁의 소셜 효과**: 직접 플레이어가 트리거하는 그룹 대 그룹 이벤트 → 소셜 결속 최고조

**최소 인원 설계 원칙 (업계 관찰 데이터):**
- Survivor.io 클랜: 최대 20명 (레벨 1 기준)
- Capybara Go 길드: 레벨 1에서 30명, 길드 레벨당 +2슬롯
- 소규모(10명 이하) 길드는 이벤트 참여 임계값 충족 어려워 비활성화 위험
- 적정 초기 규모: **20~30명** (코어 모바일 기준)

**핵심 기능 우선순위 (단계별 도입):**
1. 기초: 길드 채팅 + 멤버 목록 + 길드 포인트 기여
2. 중급: 길드 퀘스트/공동 목표 + 길드 상점
3. 고급: 길드 전쟁 + 협동 레이드

### 1-2. 소셜 결속 = 장기 리텐션 데이터 증거

**Kongregate 포트폴리오 분석 (GDC Europe 2013, 대규모 F2P 게임 데이터):**
- 길드 멤버의 수익이 비멤버 대비 **10~20배** 높음
- 경쟁 길드 기능 도입은 매출의 변곡점(inflection point)
- Kongregate 상위 게임 **100%**가 길드 보유

**Dawn of the Dragons 전환율 데이터:**
- 비길드 유저 결제 전환율: **3.2%**
- 길드 유저 결제 전환율: **23%**
- 차이: **7.2배**

**산업 전반 데이터 (다수 출처 합산):**
- 소셜 기능 도입 후 참여 및 리텐션: **2~2.5배** 향상
- 커뮤니티 소속감으로 인한 장기 플레이 가능성: **2.7배**
- 길드 참여가 ARPU에도 긍정적: Tyrant Unleashed ARPU 비교에서 길드 유저 우세

**왜 길드가 이렇게 작동하는가? (Kongregate 분석):**
1. 사회적 복귀 유인 — 친구 보고 싶음
2. 사회적 복귀 압력 — 팀을 실망시키고 싶지 않음
3. 구매 심리 개선 — "나를 위해"가 아닌 "팀을 위해" 결제
4. P2W 역학 전환 — 과금자와 비과금자 모두 팀에 가치 기여

### 1-3. 소셜 시스템을 "초기부터" 설계해야 하는 이유

**출처: design-philosophy.md §11-1, 396종 게임 분석 법칙 3번 ("소셜 레이어 필수")**

> **법칙 3**: 소셜 결속 없이 3년 생존 사례 극소수. 초기부터 설계할 것.

**"나중에 추가"의 문제 (업계 관찰):**
- 기존 유저의 핵심 플레이 루프가 이미 소셜 없이 형성됨 → 소셜 기능이 부자연스러운 추가물로 느껴짐
- 게임 아키텍처가 소셜 레이어를 수용하지 못해 반쪽 구현
- 기존 유저가 "이미 혼자 하는 게임"으로 인식 고착 → 길드 가입률 저조
- GameRefinery 관찰: 캐주얼 게임이 길드 기능을 추가 도입 시 "단계적 도입 전략" 필수 — 기초 먼저, 복잡한 기능 나중

**설계 시점의 중요성:**
- 성장 시스템에 길드 기여 요소 포함 (예: 길드 기부 → 길드 포인트 → 성장 재화)
- BM에 길드 연계 요소 포함 (예: 길드 상점 전용 화폐)
- 맵/진행 구조에 협동 콘텐츠 슬롯 확보

### 1-4. 소셜 기능의 순차적 해금 타이밍

**출처: GameRefinery 온보딩 연구, 업계 관행 분석**

| 타이밍 | 해금 기능 | 이유 |
|--------|----------|------|
| **Day 1 (FTUE 완료 직후)** | 친구 목록, 소셜 로그인, 초대 코드 | 초기 사회적 연결고리 형성 — 이탈 방지 |
| **Day 1~3** | 길드 가입 기능 | 핵심 소셜 고리를 빨리 체험시켜야 D7 리텐션 확보 |
| **Day 3~7** | 길드 채팅, 기부, 간단한 길드 퀘스트 | 루틴 형성 전 소셜 루프 완성 |
| **Day 7+** | 길드 전쟁, 협동 레이드, 경쟁 길드 이벤트 | 기초 게임 이해 후 복잡한 소셜 기능 도입 |

**핵심 원칙:** "소셜을 너무 늦게 해금하면 유저가 솔로 습관을 이미 형성"
- Day 7 이후 첫 길드 가입을 유도하는 게임은 가입률이 낮고 활성도도 낮음
- Day 1~3 이내에 적어도 "길드 가입"을 경험시키는 것이 최선
- 단, FTUE 완료 전 소셜 기능 노출은 혼란 유발 — 코어 루프 이해 후 단계적 해금

### 1-5. 소셜 압력(Social Pressure)의 양면성

**출처: ResearchGate "Pressure to play: Social pressure in online multiplayer games", 다수 업계 분석**

**긍정면 — 리텐션 강화:**
- 길드 이벤트 참가 의무감 → 주기적 복귀
- 팀원을 실망시키고 싶지 않은 심리 → 일일 접속
- 리더/임원 역할 부여 → 슈퍼 인게이지 유저 생성

**부정면 — 번아웃 위험:**
- 과도한 일일 기부 의무 + 출석 체크 + 길드 이벤트 강제 → "게임이 두 번째 직장이 됨"
- 연구: 유저 평균 주당 20시간 플레이, 상당수가 "의무감"으로 표현
- 끊임없는 콘텐츠 공급 → "완료"의 느낌 없음 → 번아웃의 새 형태
- 길드 의무가 게임 스타일과 맞지 않을 때 → 이탈 가속

**설계 원칙 — 균형 잡기:**
- 길드 참여는 **선택적 보너스**로 설계 (비참여자도 코어 게임 진행 가능)
- 일일 기부/체크인의 부담을 최소화 (시간 < 1분)
- "가벼운 소셜"부터 단계적 도입 — 처음부터 강한 의무 부여하지 않기
- 다양한 참여 강도 허용: 캐주얼 길드 vs 경쟁 길드 선택 가능하게

### 1-6. 친구 초대 시스템의 효과와 설계

**출처: GameRefinery "Power-up UA with a Friend Referral Program", 2022 연구 데이터**

**핵심 데이터:**
- 친구 초대 전환율: **30%** (일반 마케팅 채널 대비 높음)
- 초대받은 유저는 유기적 유입 유저보다 LTV가 높은 경향

**효과적인 레퍼럴 시스템 설계:**
- 초대자 + 피초대자 양쪽 모두 보상
- 보상 조건: 피초대자가 **일정 시간 이상 플레이** (단순 설치 아님)
- RAID: Shadow Legends — 피초대 친구의 레벨 달성 마일스톤에 따라 보상 (Epic Hero 등)
- 최대 초대 인원 제한 (예: 6명) → 희소성/특별감 부여
- 게임 내 의미있는 맥락에 초대 기회 배치 (길드 결성 시, 이벤트 시작 시)
- 소셜 게임플레이와 직접 연결: "친구를 초대하면 같이 레이드 가능" 등

## 2. PvP 시스템 설계 원칙

### 2-1. 실시간 PvP vs 비동기 PvP — 장르별 적합도

**출처: Skillz 분석, Wayline 아티클, 업계 전반 관찰**

| 구분 | 실시간 PvP | 비동기 PvP |
|------|-----------|-----------|
| **모바일 적합도** | 낮음 (세션 길이, 네트워크 의존) | 높음 |
| **개발/운영 비용** | 높음 (서버, 매칭 인프라) | 낮음 |
| **매칭 대기** | 필요 (문제 발생 위험) | 불필요 |
| **세션 유연성** | 낮음 (중도 이탈 불가) | 높음 (언제든 시작/종료) |
| **심리적 긴장감** | 높음 (상대방 실재) | 중간 |
| **적합 장르** | 배틀로얄, MOBA, 격투 | 전략SLG, 방치형, 캐주얼 |

**모바일에서 비동기 PvP가 우세한 이유:**
- 모바일 유저의 대다수는 "캐주얼" — 짧은 세션, 멀티태스킹 환경
- 실시간 대전은 중단되는 순간 패배 위험 → 높은 스트레스
- 비동기: "내 빌드/덱으로 AI화된 상대 베이스 공격" → 세션 무관
- 리텐션 관점: 비동기는 "나의 속도"에 맞음 → 번아웃 없이 지속 가능

**실시간 PvP의 경우:**
- 가장 강력한 사회적 즉시성 — "같은 화면에서 실제 상대와 경쟁"
- 강한 상호호혜성 → 리텐션 강화
- 단, 운영 비용과 인프라 투자가 전제조건

### 2-2. 비동기 PvP의 장점 상세

**출처: Wayline "Asynchronous Multiplayer: Reclaiming Time in Mobile Gaming"**

1. **매칭 대기 없음** — 즉시 시작, 즉시 종료
2. **짧은 세션 가능** — 3분~10분 내 완결
3. **서버 부담 적음** — 동시 접속자 수 비례 비용 없음
4. **균형잡힌 매칭** — 실시간 접속자 수에 영향받지 않음
5. **글로벌 매칭 가능** — 시간대 무관 (다른 지역 유저와 비동기 대전)
6. **기록/리플레이 활용** — 상대방의 "최고 결과"가 AI로 구현되어 다양한 상대 체험

**비동기 PvP 좋은 예시들 (GameRefinery "4 Great Examples of PvP Modes in Casual Games"):**
- Match Masters: 퍼즐 게임 + 1v1 비동기 PvP → 구독 출시 후 일일 수익 최고점, RPD 95% 증가
- Coin Master: 슬롯 스핀 → 공격/레이드 → 비동기 PvP의 완성형
- 비동기 PvP는 캐주얼 게임에서도 강력한 수익화 수단으로 증명됨

### 2-3. PvP 밸런싱 — P2W 격차와 매칭 시스템

**출처: 업계 관찰 종합**

**P2W 격차 문제:**
- 모바일 PvP의 가장 큰 위협: 과금 격차로 인한 무과금 이탈 → 생태계 붕괴
- 무과금/저과금 유저가 매칭에서 지속 패배 → 흥미 상실 → 이탈
- 이탈 패턴: 무과금 → 저과금 → 중과금 순으로 이탈 → 결국 고과금도 상대 없어 이탈

**매칭 설계 원칙:**
- **전투력 기반 매칭 (MMR)**: 과금 수준이 비슷한 유저끼리 매칭
- **보호막 시스템**: 초보자 구간 분리 (비과금 구간 보호)
- **Coin Master 해법**: 슬롯 랜덤성으로 과금 격차를 희석 — 운의 요소가 P2W 인식을 줄임
- **비동기 PvP 해법**: 실력/빌드 기반 → 전략 > 스탯 차이

**Mo.co (Supercell) 사례 (Naavik 분석):**
- PvP 없는 No P2W 구조로 론칭
- 다양한 PvP 모드 테스트 중 — 무엇이 유저에게 실제로 재미있는지 데이터 수집

### 2-4. PvP 보상 구조 — 순위 보상 vs 참여 보상

**출처: Google Play "Battle Passes", Deconstructor of Fun "Battle Passes Analysis"**

| 구분 | 순위 보상 | 참여 보상 |
|------|----------|----------|
| **특징** | 상위 N% 에게만 고가치 보상 | 참여 자체에 보상 지급 |
| **장점** | 강한 경쟁 동기, 과금 유발 강함 | 낮은 이탈률, 광범위한 만족 |
| **단점** | 무과금/신규 이탈, 격차 실감 | 경쟁 동기 약화 |
| **적합** | 코어 유저 많은 게임 | 캐주얼/혼합 유저층 |

**최적 설계 (혼합 구조):**
- **기본층**: 참여 보상 (모든 플레이어가 받을 수 있는 것)
- **상위층**: 순위 보상 (경쟁자를 위한 프리미엄 보상)
- **배틀패스 연동**: PvP 시즌과 배틀패스 연동 → "PvP 하면 BP 진행"
- 배틀패스 구매자는 비구매자 대비 세션 수가 현저히 높음

### 2-5. PvP가 BM에 미치는 영향

**출처: 업계 분석 종합**

**경쟁 심리 → 과금 경로:**
1. "지고 싶지 않다" → 전투력 강화 아이템/캐릭터 구매
2. "상위권에 들고 싶다" → 빠른 성장 속도권 구매
3. "길드를 위해 기여하고 싶다" → 길드 기부용 재화 구매
4. "남들과 달라 보이고 싶다" → 코스메틱(PvP 스킨, 이모트) 구매

**PvP는 가챠 수요를 자연스럽게 만듦:**
- 새 캐릭터/장비 출시 → "PvP 메타가 바뀜" → 신규 수집 동기
- 코인마스터 방식: 슬롯(랜덤성) + PvP(공격/레이드) → 스핀 수요 지속 창출

**수치 예시 (Match Masters, GameRefinery 2024):**
- PvP 포커스 캐주얼 게임 구독 도입 후 RPD(Revenue Per Download) **95% 증가**
- 일일 수익 최고점 달성

### 2-6. "PvP 없이도 장기 리텐션 가능한가?"

**출처: Slay the Spire 데이터, Archero 진화 분석**

**PvP 없는 장기 리텐션 사례:**
- **Slay the Spire**: 완전 솔로 로그라이트 → 데일리 챌린지 + 글로벌 리더보드로 "간접 경쟁" 구현
  - 24시간마다 초기화되는 시드 런 + 스코어 순위
  - 리더보드는 있지만 실시간 PvP 없음 → PC 기준 수년간 강력한 리텐션
- **Archero (초기)**: PvP 없이 출시 → 이후 PvP 추가 (Archero 2에서 PvP System 2.0)

**결론:**
- 로그라이트는 PvP 없이도 장기 리텐션 가능 (무한 런 반복 재미, 빌드 최적화)
- 단, 모바일 수익화 측면에서 PvP 레이어는 ARPU 향상에 강력한 도구
- **최적해: 솔로 코어 + 선택적 PvP 레이어** (강제하지 않지만 존재)
- design-philosophy.md 법칙 30: "순수 방치 = $5M 천장, 가챠×PvP×소셜 레이어가 수익 결정"

## 3. 로그라이트에서의 소셜/PvP

### 3-1. 로그라이트에서 가능한 소셜/PvP 형태

**비동기 PvP — "내 빌드 vs 상대 빌드" 자동 대전:**
- 로그라이트에서 가장 자연스러운 PvP 형태
- 자신의 런 결과(빌드, 장비)로 방어 측 AI를 만들어 공격/방어
- Clash of Clans 방식의 기지 방어를 로그라이트에 적용
- 실제 로그라이트 진행에 방해하지 않음 — 별도 PvP 모드로 분리

**협동 — 실시간 협동 런 (Soul Knight 방식):**
- Soul Knight: 2~4인 협동 던전 탐험 (로그라이트 + 실시간 협동)
- 약점: 서버 인프라 필요, 세션 길이 맞추기 어려움
- 장점: 강한 사회적 결속, 솔로와 다른 경험 제공

**간접 경쟁 — 주간/데일리 챌린지 랭킹 (Slay the Spire 방식):**
- 같은 시드(세팅)로 경쟁 → 빌드 전략 최적화 경쟁
- 순위 = 소셜 압박 없이 "내 기량 증명"
- 모바일 로그라이트에 가장 낮은 마찰로 도입 가능
- 주간 리셋으로 리텐션 루프 형성

**소셜 공유 — 빌드/결과 공유 (바이럴):**
- 특이한 빌드 조합, 클리어 영상 스크린샷 공유
- 커뮤니티 기반 "빌드 메타 토론" → 장기 참여
- 직접 구현 코스트 낮음 — 공유 버튼 + 스코어 카드 UI

### 3-2. Archero → Survivor.io → Capybara Go 소셜 기능 진화

**Archero (Habby, 2019):**
- 초기에는 소셜 기능 거의 없음
- 이후 업데이트로 길드, PvP 추가
- Archero 2에서 PvP System 2.0 (Best-of-3 매치)으로 진화

**Survivor.io (Habby, 2021):**
- v2.0.1 업데이트에서 클랜 시스템 도입 (후발 추가)
- 클랜 구성: 최대 20명, 클랜 포인트, 클랜 상점, 어셈블리 에어리어
- 클랜 경쟁: 2주마다 리셋되는 클랜 경쟁전 (자동 참가)
- 클랜 기여 지표: 개인 통계(승/킬/게임수)를 클랜 내 공개 → 공헌도 가시화
- Naavik 분석: "협동 실시간 런은 잠재적 미래 사이드 활동으로 검토 중"

**Capybara Go (Habby, 2024):**
- 길드: 최대 30명 (레벨 1), 길드 레벨당 +2슬롯
- 길드 레이드 + 길드 채팅 중심
- 게임 자체가 "소셜 카지노 RPG" — 코인마스터와 로그라이트의 혼합
- 핵심 루프: 룰렛 돌리기 → 랜덤 인카운터 → 소셜 공유 친화적 결과
- GameRefinery October 2024: 일본에서 폭발적 성공 → 소셜 카지노 + RPG 조합의 신장르 증명

**진화 패턴 분석:**
- Archero: 솔로 우선 → 소셜 후발 추가 (약점)
- Survivor.io: 솔로 우선 → 클랜 추가 (개선, 그러나 여전히 후발)
- Capybara Go: 소셜 카지노 DNA 내재 → 소셜이 코어 루프에 통합 (최선)

### 3-3. 슬롯머신 + PvP 가능성 (Coin Master 증명 모델)

**출처: Deconstructor of Fun "How Coin Master Disrupted Social Casino and Pocketed $100M" (2019), Udonis 분석**

**Coin Master 수익 구조:**
- 누적 다운로드 200M+
- 슬롯머신 기반 PvP (공격/레이드)
- 순수 슬롯 게임 → 소셜/PvP/건설 메타로 확장

**핵심 루프:**
```
스핀(슬롯) → 코인 획득/공격/레이드/쉴드 → 마을 건설 → 새 마을 언락 → 반복
```

**PvP 메커닉:**
- 공격: 슬롯에서 해머 3개 → 상대방 건물 파괴
- 레이드: Coin Master 아이콘 → 상대방 창고 털기 (잭팟 등가)
- 쉴드: 슬롯에서 쉴드 → 방어 소모품

**왜 이 모델이 작동하는가:**
- 슬롯 랜덤성이 P2W 격차 인식을 희석 ("운이 없었을 뿐")
- PvP가 스핀 소모를 자연스럽게 촉진 (공격하려면 스핀 필요)
- 소셜 알림: "친구가 당신을 공격했습니다" → 복귀 유인
- 페이스북 연동 → 실제 지인과의 PvP → 소셜 압박 극대화

**로그라이트 + 슬롯에의 시사점 ("운수 좋은날" 관련):**
- 슬롯(랜덤성) + 로그라이트 런 결과 → 비동기 PvP로 "내 런 빌드 vs 상대 방어" 구현 가능
- Coin Master의 공격/레이드를 "최고 런 기록 vs 상대방 기록"으로 추상화 가능
- 소셜 알림 ("누군가 당신의 런 기록에 도전했습니다") → 복귀 유인

## 4. 소셜 기능의 BM 연계

### 4-1. 길드 기부/기여 시스템 → 과금 유도

**메커니즘:**
- 길드 기부 → 길드 포인트 → 길드 레벨업 → 멤버 전체 버프
- "팀을 위해" 기부하는 심리 → 재화 구매 유도
- Kongregate 분석: 길드 내 과금 심리 = "나를 위해"보다 수용도 높음
- 기여도 순위표 → 기여 경쟁 → 추가 구매

**설계 포인트:**
- 기부 상한선 설정 (고래 독점 방지)
- 일일 기부 한도 → 매일 접속 유인
- 기부 재화 ≠ 핵심 성장 재화 (착취 느낌 방지)

### 4-2. PvP 시즌 보상 → 시즌패스 연동

**출처: Deconstructor of Fun "Battle Passes Analysis" (2022), GameRefinery 배틀패스 분석**

**핵심 데이터:**
- Top 100 게임 중 30개 이상이 배틀패스 보유
- 배틀패스 구매자 = 비구매자 대비 세션 수 현저히 높음
- PvP 미션을 배틀패스 진행 조건으로 삽입 → "PvP 해야 BP 진행"

**설계 공식:**
- 시즌 기간: 4~6주 (FOMO 압박, 그러나 번아웃 없는 적정 기간)
- 무료 트랙 + 프리미엄 트랙 이중 구조
- PvP 전용 코스메틱 보상 → PvP 참여 동기
- 시즌 스토리 + PvP 랭킹 연동 → 몰입도 강화

### 4-3. 소셜 과시(Social Flex) → 코스메틱 판매

**출처: Udonis "Cosmetic Monetization Trend", 업계 분석**

**작동 원리:**
- 희귀 코스메틱 = 플레이 헌신도/과금의 시각적 증거
- PvP에서 희귀 스킨 착용 → 상대방에게 과시 → 사회적 지위
- 길드 전쟁 시 팀 유니폼 개념의 길드 코스메틱

**수치:**
- 유저의 47%가 게임 내 30시간 이상 플레이 후 코스메틱 구매
- 5% 미만이 4시간 미만 플레이 후 구매 → 장기 리텐션이 코스메틱 판매의 전제

**"Social Flex" 설계 원칙:**
- 희귀 코스메틱은 구매만이 아닌 게임 내 업적으로도 획득 가능하게 (구매 외 경로 → 동경의 대상)
- PvP 상위권 전용 아이템 → 경쟁 동기 + 과금 동기 동시 충족
- 길드/클랜 전용 코스메틱 → 소속감 + 집단 정체성

### 4-4. 협동 콘텐츠 → 캐릭터/장비 다양화 → 가챠 수요

**메커니즘:**
- 협동 레이드: 각 역할(탱커/딜러/힐러) 필요 → 다양한 캐릭터 보유 동기
- 길드 전쟁: 상대 조합에 따른 카운터픽 필요 → 컬렉션 확장 동기
- 새 협동 콘텐츠 출시 → 새 최적 캐릭터/장비 필요 → 가챠 타이밍
- 수집형 RPG에서 소셜/PvP는 가챠 신규 콘텐츠 출시의 명분

## 5. 소셜 시스템 실패 패턴

### 5-1. 소셜이 너무 강제적 → 솔로 유저 이탈

**증거:**
- 게임 진행에 길드 의무가 필수일 때 → 솔로 선호 유저 즉시 이탈
- "길드 없으면 핵심 콘텐츠 접근 불가" 설계 → 강요의 느낌
- 모바일 유저의 상당수는 소셜 없이도 플레이하고 싶어함

**설계 해법:**
- 소셜 = 보너스 레이어 (없어도 코어 게임 완결)
- 길드 가입 유도하되 강제하지 않기
- 솔로 전용 콘텐츠와 소셜 콘텐츠를 병렬 운영

### 5-2. 길드 의무(일일 기부/출석) → 번아웃

**증거 (ResearchGate 연구, 업계 관찰):**
- 과도한 일일 의무 → 게임을 두 번째 직장으로 느낌
- 길드 의무 불이행 시 사회적 처벌(강퇴, 비난) → 독소적 환경
- 번아웃 유저: 플레이 빈도 감소 → 최종 이탈

**설계 해법:**
- 일일 기부 시간 부담 < 1분
- 길드 이벤트 참가는 선택적, 불참 페널티 없음
- "가벼운 참가"로도 충분한 기여 가능하게

### 5-3. PvP 격차 → 무과금 이탈 → 생태계 붕괴

**이탈 사이클:**
1. 강한 P2W PvP → 무과금 지속 패배
2. 무과금 이탈 → 매칭 풀 감소
3. 저과금도 대전 어려워짐 → 저과금 이탈
4. 고과금만 남음 → 상대 없어 고과금도 이탈
5. 게임 생태계 붕괴

**설계 해법:**
- MMR 기반 매칭으로 전투력 유사 유저끼리 대전
- 보호 구간 설계 (신규/무과금 전용 매칭 풀)
- 슬롯 랜덤성 활용으로 P2W 인식 희석 (Coin Master 모델)
- 참여 보상 구조 → "지더라도 보상 있음"

### 5-4. 매칭 실패(대기 시간 3분+) → 이탈

**출처: 매칭 연구 데이터**

**수치:**
- 플레이어는 대기 시작 후 수 분 이내에 이탈 결정
- 진행 표시 없으면 "게임이 고장난 것" 인식 → 즉시 이탈
- 30초~5분이 일반적 허용 범위, 그 이상은 이탈 급증

**설계 해법:**
- 대기 중 프로그레스 바 + 예상 시간 표시
- 대기 중 다른 활동 가능하게 (상점 둘러보기, 캐릭터 확인 등)
- 서버 풀이 작을 때 비동기 PvP로 전환하거나 AI 상대 제공
- 실시간 PvP를 시도하기 전 DAU가 충분한지 확인

### 5-5. "나중에 추가"한 소셜 → 기존 유저 무관심

**관찰 패턴:**
- 기존 유저는 이미 솔로 플레이 루틴 형성
- 소셜 기능이 기존 성장 경로와 단절 → 부자연스러운 추가물
- 구조적 한계: 게임 경제/밸런스가 소셜 보상을 수용하지 못함
- Survivor.io 사례: 클랜 v2.0.1 이후 추가 → 코어 루프와의 통합 한계

**설계 해법:**
- 소셜 레이어는 게임 설계 초기부터 포함
- 성장 시스템 → BM → 이벤트 구조 모두 소셜 레이어를 전제로 설계

## 6. 장르별 소셜/PvP 설계 가이드

### 6-1. 로그라이트 장르 최적 소셜/PvP 형태 (종합)

**낮은 마찰, 높은 효과 순서:**

| 우선순위 | 형태 | 구현 난이도 | 리텐션 효과 | 수익 효과 |
|---------|------|-----------|------------|----------|
| 1 | 주간/데일리 챌린지 랭킹 | 낮음 | 중간-높음 | 낮음 |
| 2 | 빌드 공유 + 스코어카드 | 낮음 | 중간 | 낮음 |
| 3 | 길드 (선택적) + 길드 채팅 | 중간 | 높음 | 중간 |
| 4 | 비동기 PvP (빌드 vs 빌드) | 중간 | 높음 | 높음 |
| 5 | 길드 전쟁 + 협동 레이드 | 높음 | 매우 높음 | 매우 높음 |
| 6 | 실시간 협동 런 | 매우 높음 | 매우 높음 | 높음 |

### 6-2. 슬롯머신 + 로그라이트 조합("운수 좋은날")에서의 시사점

**Coin Master 모델의 로그라이트 적용:**
- 슬롯 스핀 결과 = 비동기 PvP의 랜덤 강화 요소로 활용
- "스핀 → 런 결과 강화 → 더 좋은 런 기록 → PvP 방어력 향상"
- 슬롯 랜덤성이 P2W 격차를 심리적으로 희석 → 공정한 PvP 인식

**최소 도입 세트 (우선순위):**
1. 주간 챌린지 랭킹 (즉시 구현 가능, 리텐션 효과 즉각)
2. 빌드/결과 공유 버튼 (바이럴 효과)
3. 길드 (선택적) — 기부 + 길드 퀘스트
4. 비동기 PvP — "내 최고 런 기록으로 방어 세트업"

## 7. 핵심 원칙 체크리스트 (소셜/PvP 설계 시 필수 확인)

### 소셜 시스템
- [ ] 소셜 레이어가 초기 설계에서 포함되어 있는가? (나중 추가 금지)
- [ ] 길드 가입은 선택적이고 솔로 유저도 코어 게임 완결 가능한가?
- [ ] 소셜 해금 타이밍이 Day 1~3 이내인가? (D7 이후 첫 해금은 늦음)
- [ ] 일일 길드 의무 시간이 1분 미만인가?
- [ ] 친구 초대 시스템이 양측 보상 구조를 갖추고 있는가?
- [ ] 길드 기능이 BM과 연결되어 있는가? (길드 상점, 기부 등)

### PvP 시스템
- [ ] 장르와 유저층에 맞는 PvP 형태(실시간 vs 비동기)인가?
- [ ] P2W 격차를 완화하는 매칭 시스템이 있는가?
- [ ] 참여 보상이 순위 보상과 함께 설계되어 있는가?
- [ ] 매칭 대기 시간이 2분 이내인가? (혹은 비동기로 처리)
- [ ] PvP가 시즌패스/배틀패스와 연동되어 있는가?
- [ ] 무과금 유저도 PvP에서 최소한의 재미를 느낄 수 있는 구조인가?
