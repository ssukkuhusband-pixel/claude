## 1. 모바일 전투의 물리적 제약

### 1-1. 터치스크린 입력의 3대 한계

| 한계 | 내용 | 설계 대응 |
|------|------|-----------|
| **손가락 가림(Occlusion)** | 화면 중앙~상단이 엄지손가락에 가려짐 | 주요 UI/버튼을 화면 하단 30% 이내에 배치 |
| **정밀도 부재** | 터치 포인트가 픽셀 단위 정밀도 불가 | 히트박스를 시각적 오브젝트보다 20~40% 크게 설계 |
| **물리 피드백 부재** | 키보드/패드의 클릭감 없음 | 햅틱 피드백(진동) + 시각/청각 피드백으로 보완 |

**출처**: MobileFreeToPlay.com 터치 조작 설계 분석 (2024), Alibaba Product Insights

### 1-2. 조작 스탠스별 유저 분포와 설계 함의

모바일 유저는 크게 3가지 스탠스로 게임을 함다:

```
[조작 스탠스 분류]

1. 게이머 스탠스 (양손 엄지) — 코어 게이머 소수
   └ 장점: 정밀 조작 가능
   └ 단점: 화면 중앙 상단이 엄지에 가려짐
   └ 설계 대상: 미드코어/코어 게임

2. 캐주얼 프로더 (한 손 고정 + 다른 손 터치) — 대다수 유저
   └ 장점: 가장 광범위한 시장
   └ 단점: 두 개 동시 입력 불가 (한 손이 기기 고정에 사용)
   └ 설계 대상: 캐주얼~미드코어, 광범위한 도달 원할 때

3. 지하철 엄지 (한 손으로 모두) — 이동 중 유저
   └ 장점: 통근/이동 중 플레이
   └ 단점: 화면 하단 좁은 구역만 안정적 조작 가능
   └ 설계 대상: 방치형, 턴제, 단순 탭 조작 게임
```

**핵심 인사이트**: Angry Birds, Fruit Ninja, Cut the Rope의 공통점 — 모두 "캐주얼 프로더"에 최적화되어 가장 광범위한 유저에게 도달함다. **한 손 캐주얼 설계 = 시장 규모 극대화**.

**출처**: MobileFreeToPlay.com "Touch Control Design: Ways Of Playing On Mobile"

### 1-3. 한 손 vs 양손 — 수익 함의

| 조작 방식 | 도달 가능 유저 | 조작 복잡도 | 성공 장르 사례 |
|-----------|---------------|-------------|----------------|
| **원터치(한 손)** | 최대 (casual~midcore) | 낮음 | 퍼즐, 방치형, 턴제 가챠 |
| **양손 탭** | 중간 | 중간 | 수집형 RPG, 전략 |
| **가상 조이스틱(양손)** | 코어만 | 높음 | MOBA, 액션RPG |

**결론**: 순수 수익만 보면 양손 조작 코어 게임이 ARPU는 높지만 도달 유저가 좁음. 캐주얼~미드코어 한 손 설계가 볼륨×수익의 최적점임다.

### 1-4. 세션 길이 적정 기준

**GameAnalytics 2025 벤치마크 데이터 (전체 모바일 게임)**:

| 성과 티어 | 평균 세션 길이 | 일일 세션 수 |
|-----------|--------------|-------------|
| **상위 25%** | 8~9분 | 6~7회 (미드코어) |
| **중앙값** | 5~6분 | 4회 |
| **하위 티어** | 3~4분 | — |
| **중앙값 일일 플레이 시간** | 22분 | — |

**전투 설계 적용 기준**:
- **1회 전투 적정 시간**: 1~3분 (세션 5~6분 내 2회 전투 완료 구조)
- **중동/아시아 유저**: 세션 5분 이하 선호 → 전투 1~2분 구조 필수
- **미드코어 하드코어 스테이지**: 3~5분 허용 (단, 자동완료/스킵 옵션 병행)

**방해받는 환경 설계 원칙**:
- 전투 도중 앱 전환 시 상태 보존 필수 (서버 기반 세이브 스테이트)
- 1회 전투가 **완결된 의미 있는 단위**여야 함 — 중간에 끊겨도 "이번 판은 다음에" 가능한 구조
- 긴 전투가 필수인 경우 → 오토 전투 + 배속 기능으로 "방치 후 회수" 구조 제공

**출처**: GameAnalytics Mobile Gaming Benchmarks 2025

## 2. 전투에서의 재미의 공식

### 2-1. 성취감 요소 (Mastery / Achievement)

전투에서 성취감을 만드는 핵심 메커니즘:

```
[전투 성취감 피라미드]

최상위 (강한 성취감)
├── 오랜 시간 막혔던 스테이지 클리어
├── 완벽한 전략 조합 발견 (팀 시너지 극대화)
├── 상위 PvP 랭킹 진입
└── 희귀 캐릭터 풀파워 달성

중간 (일상적 성취감)
├── 콤보 성공 / 크리티컬 연속 히트
├── 클리어 타임 단축 / 무피해 클리어
├── 새 몬스터/장비 획득
└── 스킬 레벨 업

일상 (소소한 성취감)
├── 스테이지 클리어 (첫 3성)
├── 골드/재화 획득
└── 데일리 퀘스트 완료
```

### 2-2. 의외성 요소 (Surprise / Variability)

전투에서 의외성을 만드는 핵심 메커니즘:

| 메커니즘 | 설명 | 성공 사례 |
|----------|------|-----------|
| **랜덤 크리티컬** | 기댓값 있지만 타이밍 예측 불가 | 서머너즈워 크리티컬 시스템 |
| **보스 패턴 변화** | 체력 구간별 다른 공격 패턴 | Puzzle & Dragons 던전 설계 |
| **랜덤 보드** | 매 전투 다른 퍼즐 배치 | P&D 오브 매칭 시스템 |
| **팀 시너지 발견** | 예상치 못한 조합 효과 폭발 | 서머너즈워 룬+몬스터 조합 |
| **로그라이트 랜덤 선택** | 매 런마다 다른 능력 조합 | 궁수의전설, 카피바라Go |
| **FOMO 한정 이벤트** | 기간 한정 컨텐츠로 긴장감 | 대부분의 가챠 게임 |

**핵심 원칙**: 의외성은 **완전한 무작위**가 아니라 **통제 가능한 변수 범위 내의 랜덤**이어야 함다. 완전 랜덤 = 플레이어 에이전시 소멸 → 허무감.

### 2-3. "같은 전투 반복해도 안 질리는" 게임의 비밀

**퍼즐&드래곤 (10년+ 서비스) 사례 분석**:

P&D가 같은 매치-3 전투를 10년 넘게 유지할 수 있었던 이유:

1. **숙련도 곡선의 무한 확장**: 초보는 "최대한 매칭"이지만, 고수는 "정확한 경로로 최대 콤보 설계". 같은 메커닉이지만 숙련도에 따라 **완전히 다른 게임**이 됨
2. **팀 빌딩의 무한 조합**: 강한 팀 = "가장 강한 몬스터 5마리"가 아니라 속성 커버리지 + 생존력 + 콤보 발생률의 최적화. 수십만 가지 팀 조합이 모두 다른 플레이를 만듦
3. **던전 설계의 다양성**: 고정된 적 패턴 → 플레이어가 대응 전략을 설계 → 전략적 사고의 즐거움
4. **콜라보레이션으로 지속적 신선함**: Monster Hunter, Evangelion, Dragonball Z 등 150개+ 협력. 같은 전투 메커닉에 새로운 캐릭터/미션을 덮어씌워 신선함 제공

**서머너즈워 (10년+ 서비스, 누적 매출 $3B+) 사례 분석**:

1. **룬 시스템이 진짜 게임**: 23가지 룬 세트 × RNG 서브스탯 = 사실상 무한한 최적화 목표
2. **PvP 생태계의 메타 변화**: 신규 몬스터 추가 시 PvP 메타 변화 → "이번 달 최강 조합" 계속 바뀜
3. **2014년 출시 몬스터가 2024년 PvP에서도 통용**: 초기 투자(몬스터 육성)의 가치 보존 → 장기 유저 이탈 방지

**출처**: GameRefinery "10 Years of Puzzle & Dragons", Game Developer P&D Monetization 분석, Summoners War 10주년 인터뷰 (Pocket Gamer)

### 2-4. 시각/음향 피드백 (Game Feel / Juice)

**"Juice"의 정의**: 플레이어 액션에 대한 상시적이고 풍부한 피드백 설계. 메커닉은 같아도 피드백의 강도와 품질이 **체감 재미를 결정**함다.

**전투 피드백 4대 요소**:

| 요소 | 설계 원칙 | 예시 |
|------|-----------|------|
| **시각적 큐** | 히트 중요도에 비례한 시각 보상. 크리티컬=빨강, 일반=노랑, 미스=회색 | 데미지 숫자 컬러 코딩 |
| **청각 피드백** | 타격음+크리티컬음의 도파민 트리거. 콤보 연속 히트 시 음량/음색 상승 | 랜덤크리티컬 타격음 |
| **햅틱 피드백** | 중요 이벤트(크리티컬, 스킬 사용)에만 선택적 진동. 과다 사용 금지 | iOS/Android 햅틱 엔진 활용 |
| **스크린 효과** | 히트 시 화면 흔들림, 슬로우모션, 줌인 등 연출 | 보스 피격 시 연출 강조 |

**핵심 심리 메커니즘**: 크리티컬 히트음 = 도파민 분비 트리거. 행동(공격) → 보상 예측(크리티컬 가능?) → 사운드/시각 확인(크리티컬!) → 도파민 강화. 이 사이클이 반복 플레이의 심리적 동력임다.

**출처**: SpeeQual Games "Psychology Behind Game Audio Feedback", Acagamic "Juicy damage UI feedback"

## 3. 전투 복잡도와 접근성의 균형

### 3-1. 조작 복잡도 5단계 성공 사례 분류

```
복잡도 1 — 원터치/탭
├── 예시: 쿠키런: 킹덤(자동전투+편성), 방치형 RPG
├── 장점: 최대 도달 유저, 방해 환경에서도 플레이 가능
├── 단점: 전략 깊이 한계, ARPU 낮은 경향
└── 수익 모델: 광고 + 경량 가챠

복잡도 2 — 방향 스와이프/드래그
├── 예시: Puzzle & Dragons(오브 드래그), Reigns(카드 스와이프)
├── 장점: 단순하지만 숙련도 무한 확장 가능
├── 단점: 설계 어려움 (단순 조작으로 깊이 만들기)
└── 수익 모델: 가챠 + 스태미나

복잡도 3 — 탭+선택 조합 (턴제)
├── 예시: 서머너즈워, AFK Arena, 원신(모바일)
├── 장점: 코어/캐주얼 모두 만족, BM 연결 최적
├── 단점: 오토 전투 필수로 실제 전략성 희석될 수 있음
└── 수익 모델: 가챠 + BattlePass + PvP

복잡도 4 — 실시간 멀티액션
├── 예시: 원신(PC/콘솔 수준 조작), 모바일 MOBA
├── 장점: 높은 몰입감, 코어 유저 잡음
├── 단점: 터치 정밀도 한계로 PC/콘솔 대비 열화, 도달 유저 협소
└── 수익 모델: 코스튬+배틀패스 (P2W 지양)

복잡도 5 — 콘솔급 실시간
├── 예시: 모바일 격투게임, 고사양 액션 RPG
├── 단점: 모바일에서 물리적 불가 수준. 대부분 실패
└── 결론: 모바일에서 지양해야 할 복잡도
```

### 3-2. "쉽게 배우고 어렵게 마스터하는" 설계의 구체적 구현

**P&D 사례 — 조작 1분 이해, 마스터리 수년**:
- **배움**: 오브를 드래그해서 같은 색 3개 이상 매칭
- **마스터리**: 한 번의 드래그 경로로 5~6콤보 동시 달성하는 경로 계산

**서머너즈워 사례**:
- **배움**: 몬스터 4마리 편성 → 자동전투 ON
- **마스터리**: 23가지 룬 세트 × 6슬롯 × RNG 서브스탯 최적화 → 사실상 끝없는 최적화

**핵심 원칙**: "쉬운 입문 + 어려운 마스터리"는 **별도의 복잡한 시스템**을 추가하는 게 아니라, **같은 메커닉에서 숙련도 층위를 다층적으로 설계**하는 것임다.

### 3-3. 자동 전투(오토)의 역할

**오토 전투가 필요한 이유**:
- 반복 파밍 구간에서 시간 효율성 제공 (유저 시간 존중)
- 세션 방해 환경 대응
- 초보 유저의 진입 장벽 완화

**오토 전투가 ON이어야 하는 구간**:
- 이미 클리어한 스테이지 파밍
- 데일리 루틴 (소탕/반복 던전)
- 방치 구간

**오토 전투가 OFF여야 하는 구간 (= 플레이어가 직접 조작해야 할 때)**:
- 첫 클리어 시도 (전략적 도전)
- PvP 실시간 대전
- 레이드/협력 콘텐츠
- 스토리 보스전 (연출 체험)

**오토 전투의 BM 연결**:
- 자동 전투 속도 제한 (무료 1x, 과금 2x~3x) → 시간 판매
- 자동 소탕 티켓 제한 → 프리미엄 멤버십 판매
- 배속 기능 배틀패스 포함 → 강력한 구독 유인

**출처**: ultimategacha.com "Best Gacha Games with Auto-Battle 분석", AFK Arena 설계 분석

## 4. 전투와 BM의 연결

### 4-1. 전투 시스템이 자연스럽게 과금을 유도하는 구조

**모바일 전투 BM의 3가지 축**:

```
[모바일 전투 BM 구조]

① 전력 판매 (Power Monetization)
   ├── 강한 캐릭터/장비 가챠
   ├── 스킬 강화 재화
   └── 룬/장비 최적화 재료
   → 전투에서 "명확히 더 강하다"는 체감이 필수

② 시간 판매 (Time Monetization)
   ├── 스태미나 회복
   ├── 소탕/자동 진행 속도
   └── 대기 시간 단축
   → 전투 자체는 무료지만, 더 자주 전투하려면 과금

③ 편의 판매 (Convenience Monetization)
   ├── 배속 기능 (배틀패스 포함)
   ├── 팀 슬롯 확장
   └── 인벤토리 확장
   → 게임 경험 품질 향상, 비P2W 친화적
```

**P&D 과금 구조 (월 $100M+ 수익)**:
- 레어 몬스터 가챠: $5 / 횟수
- 몬스터 슬롯 확장: $1
- 스태미나 회복: $1
- 던전 내 컨티뉴: $1
- **월 $20 상당 무과금 프리미엄 재화 제공** → 전체 유저가 프리미엄 경제 참여

**GungHo의 성공 공식**: "공정성 + 지속적 변화 + 관대함". 숙련도가 운과 과금을 어느 정도 보상할 수 있는 구조 → 유저 존중 → 장기 잔존율 높음 → 결국 총 수익 극대화.

**출처**: Game Developer "P&D Monetization: How Great Game Design Drives GungHo's Global Hit"

### 4-2. "이 캐릭터/장비가 전투에서 확실히 다르다"는 체감의 중요성

**가챠 캐릭터 체감 차별화 3원칙**:

1. **스킬 메커니즘의 질적 차별**: 수치가 10% 더 높은 게 아니라, 아예 다른 메커니즘을 가져야 함다. 예) 기존 공격형 캐릭터 vs 신규 "적 버프 흡수 + 역이용" 캐릭터
2. **시각적 임팩트 차이**: 고등급 캐릭터의 스킬 연출은 저등급과 **눈에 띄게 달라야 함다**. 이것이 "갖고 싶다"는 욕구의 핵심 트리거
3. **전략적 가능성 개방**: 신규 캐릭터가 기존에 불가능했던 팀 조합이나 클리어 전략을 열어줄 때 — "이 캐릭터 없으면 불가능한 뭔가"가 있을 때 강력한 과금 유인

**Summoners War 성공 원인**: 2014년 출시 몬스터가 2024년 PvP에서도 통용됨 → 초기 투자의 가치 보존 → 장기 유저 이탈 방지 + 신규 유저도 "예전 캐릭터 키우면 PvP 가능"이라는 인식 → 진입 장벽 완화

### 4-3. 전투 메커닉과 가챠의 연결

**가챠와 잘 연결되는 전투 구조**:

| 전투 타입 | 가챠 연결 방식 | 과금 트리거 | 사례 |
|-----------|--------------|-------------|------|
| **턴제 수집형** | 캐릭터별 스킬 차이 극명 | "이 캐릭터가 있으면 클리어 가능" | 서머너즈워, P&D |
| **자동 방치형** | 팀 전력 수치화 (전투력) | "전투력 X 이상이면 오토 가능" | AFK Arena |
| **실시간 액션** | 캐릭터별 조작감 차이 | "이 캐릭터 조작이 재밌다" | 원신 |
| **퍼즐 기반** | 캐릭터가 퍼즐 해법에 영향 | "이 몬스터 있으면 클리어 전략이 생긴다" | P&D |

**FOMO(Fear of Missing Out) 메커니즘**:
- 한정 기간 캐릭터 → "지금 뽑지 않으면 영원히 못 뽑음"
- 새 캐릭터가 현재 레이드/이벤트에 최적화 → "지금 없으면 이번 달 컨텐츠가 힘들다"
- 이 두 가지가 결합할 때 지출 트리거 최대화

**출처**: MDPI "Addiction and Spending in Gacha Games" (2024), Gacha Game Wikipedia 분석

### 4-4. PvP 전투에서의 과금 유도 구조

**PvP 과금의 설계 딜레마**:

```
P2W(Pay-to-Win) 구조:
├── 단기 수익: 높음 (코어 과금 유저 스펜딩 극대화)
├── 장기 건강성: 낮음 (무과금 유저 이탈 → 과금 유저도 떠남)
└── 결론: 단기 이익, 장기 게임 수명 단축

공정한 PvP 구조 (Pay-to-Progress):
├── 단기 수익: 중간
├── 장기 건강성: 높음 (경쟁 환경 유지)
└── 결론: 지속 가능한 장기 수익
```

**고래(Whale) 유저 지출 패턴 (deltaDNA 데이터)**:
- 전형적 거래 규모: **$20** (소액 반복 결제)
- 54% 이상의 고래가 **단일 거래 $50 미만**
- 연간 $1,000+ 지출 고래도 평균 55건 거래 (소액 반복)
- **시사점**: 고가 단건 번들 전략 < 중간 가격 반복 구매 유도 전략

**PvP 과금 유도 최적 구조**:
1. **시즌 시스템**: 시즌 보상(코스튬, 칭호)에 과금 요소 배치 → 경쟁 기간 한정 소비
2. **전력 상한선**: 과금으로 살 수 있는 최대 전력을 제한 → 무과금도 최강 경쟁 가능성 유지
3. **편의 과금**: 스태미나, 배속, 도전권 → P2W 아닌 P2Convenience

**출처**: Game Developer "New Insights Into Spending Patterns of Whales", Wayline.io P2W 분석

## 5. 전투 메커닉의 수명

### 5-1. 10년+ 서비스 게임들의 전투 공통 패턴

| 게임 | 핵심 전투 메커닉 | 10년 생존 비결 | 누적 매출 |
|------|----------------|----------------|-----------|
| **Puzzle & Dragons** | 드래그 오브 매칭 | 무한 숙련도 + 팀 빌딩 깊이 + 협력 콜라보 | $7B+ |
| **Monster Strike** | 4인 협동 구슬 튕기기 | 협동 메커닉의 소셜 결속 + 일본 문화 특화 | $7.2B (역대 1위) |
| **Summoners War** | 턴제 + 룬 최적화 | PvP 메타 순환 + 초기 투자 가치 보존 | $3B+ |
| **Clash of Clans** | 기지 공격 전략 | 길드 전쟁 소셜 구조 + 지속적 새 유닛 | $15B+ |

**공통 패턴 추출**:

1. **핵심 메커닉 불변**: 10년 동안 전투의 핵심 입력 방식은 변하지 않음. 오브 드래그는 오브 드래그, 구슬 튕기기는 구슬 튕기기
2. **깊이는 외부에서 확장**: 팀 빌딩, 룬 최적화, PvP 메타, 콜라보 캐릭터 등 **전투 주변 시스템**이 진화
3. **소셜 레이어 필수**: 4개 게임 모두 강력한 길드/협동/PvP 구조 보유 → 이탈 방어막 역할
4. **플레이어 투자 가치 보존**: 2014년 육성한 캐릭터가 2024년에도 통용 (SW) → 장기 유저 이탈 방지

### 5-2. 전투가 질리기 시작하는 시점과 대응

**질리기 시작하는 신호**:
- D30 리텐션 급락 (3% 미만으로 떨어질 때)
- 오토 전투 사용률 90%+ (수동 전투의 매력 소진)
- 커뮤니티에서 "메타가 굳었다" 반응

**대응 전략 비교**:

| 전략 | 내용 | 장단점 |
|------|------|--------|
| **콘텐츠 확장** | 새 스테이지, 새 적, 협력 이벤트 추가 | 개발 비용 지속 발생, 빠른 소비 |
| **캐릭터/메타 순환** | 신규 캐릭터로 PvP 메타 주기적 변화 | 낮은 개발 비용, 가챠 수익과 연결 |
| **시스템 레이어 추가** | 룬, 각성, 전수 등 새 성장 축 추가 | 복잡도 증가 리스크, 장기 흥미 유지 효과 |
| **소셜 이벤트** | 길드전, 시즌 이벤트 | 소셜 결속 강화, 재방문 유도 |

**P&D의 정답**: 핵심 메커닉 불변 + 콜라보레이션(신선한 캐릭터) + PvP 모드 추가(2021) + 팀 편성 깊이 확장 (세 가지 동시)

**서머너즈워의 정답**: 핵심 메커닉 불변 + 신규 몬스터 주기적 추가(PvP 메타 순환) + 룬 시스템 심화 + 월드 아레나 실시간 PvP 도입

### 5-3. 전투 메커닉 업데이트 없이 콘텐츠만으로 연장하는 방법

**효과적인 콘텐츠 연장 구조 (메커닉 변경 없이)**:

1. **이벤트 던전**: 기간 한정 고난이도 던전 (클리어 = 희귀 보상) → 반복 도전 유인
2. **시즌 시스템**: 3개월 단위 시즌별 전투 환경 변화 (밸런스 조정)
3. **협력 콘텐츠**: 기존 전투에 멀티플레이 레이어 추가 → 소셜 결속 강화
4. **챌린지 모드**: 동일 스테이지에 "제약 조건" 추가 (ex: 단색 팀만 허용)

**콘텐츠만으로 연장할 때의 한계**:
- 결국 유저는 "같은 전투를 반복"한다는 사실을 알아차림
- D90+ 장기 리텐션은 소셜 레이어 없이는 한계 존재
- 신규 유저 유입이 줄면 콘텐츠 소모 속도 가속

**결론**: 콘텐츠 연장은 단기 유효, 장기 생존은 반드시 소셜/PvP 구조 필요

## 6. 성공 패턴 종합

### 6-1. 모바일에서 성공하는 전투의 6대 공통 패턴

```
패턴 1. 조작 단순성 + 전략 깊이의 공존
└── 입문: 10초 이내 이해 가능
└── 마스터리: 수백 시간의 최적화 여지
└── 메커니즘: 같은 조작, 다른 의사결정 층위

패턴 2. 시각/청각 피드백의 강도
└── 타격감 > 전투 밸런스
└── "공격이 꽂히는 느낌" 없으면 아무리 시스템이 좋아도 체감 재미 없음
└── 크리티컬/콤보 사운드+비주얼이 반복 플레이의 심리적 동력

패턴 3. 세션 독립성
└── 1전투 = 1완결 경험 (중간에 끊겨도 의미 있는 단위)
└── 평균 1~3분으로 설계
└── 긴 전투가 필요하면 오토+배속+소탕으로 "방치 가능" 구조

패턴 4. 반복 플레이에서의 변화
└── 매 전투 랜덤 요소 (크리티컬, 보드, 드랍) 최소 1개 이상
└── 팀 빌딩/룬 최적화로 "같은 적을 다른 방식으로 클리어" 가능
└── 신규 캐릭터 추가로 주기적 메타 변화

패턴 5. BM과의 자연스러운 연결
└── 전투 → 막힘 → "이 캐릭터 있으면 가능" → 가챠 충동
└── 과금이 전투력 향상으로 **즉각 체감**되어야 함
└── 단, P2W 구조는 장기 커뮤니티 파괴 → 공정한 과금 설계 필수

패턴 6. 소셜 레이어와의 통합
└── PvP: "내 전투 결과가 타인에게 영향" → 경쟁 동기 강화
└── 길드: 협동 전투 → 소속감 + 이탈 방어막
└── 소셜 없이 D90+ 장기 리텐션 유지한 전투 게임 사례 극소수
```

### 6-2. 전투 설계 실패 패턴 (데이터 기반)

| 실패 패턴 | 증상 | 원인 |
|-----------|------|------|
| **터치 조작 과복잡** | 초기 이탈률 폭증 | D/Pad+버튼 5개 이상 동시 조작 요구 |
| **전략 깊이 없는 단순 반복** | D30 급락 | "강하면 이기는" 단순 구조 → 숙련도 천장 너무 낮음 |
| **즉각 피드백 부재** | 체감 재미 낮음 | 타격음 약함, 데미지 숫자 작음, 이펙트 밋밋함 |
| **세션 너무 긴 전투** | 방해 환경 이탈 | 1전투 5분+ → 전화/알림에 이탈 발생 |
| **P2W PvP** | 커뮤니티 분열 | 과금 유저만 승리 → 무과금 이탈 → 과금 유저도 이탈 |
| **오토 전투 미제공** | 피로도 급증 | 이미 클리어한 스테이지 반복에 수동 조작 강요 |

### 6-3. 참치김밥 전투 시스템 설계 체크리스트

전투 시스템을 새로 설계하거나 검토할 때 사용하는 체크리스트임다:

**[기본 설계 검증]**
- [ ] 1회 전투 시간이 1~3분 이내임까? (5분+ 이면 오토+소탕 옵션 필수)
- [ ] 한 손으로도 기본 조작 가능함까? (캐주얼 프로더 지원)
- [ ] 주요 버튼/UI가 화면 하단 40% 이내에 배치되어 있슴까?
- [ ] 세션 중단 시 상태 보존이 되슴까?

**[재미 검증]**
- [ ] 10초 안에 조작 방식을 이해할 수 있슴까?
- [ ] 숙련도에 따라 전투 결과가 달라지는 의사결정 층위가 있슴까?
- [ ] 매 전투 랜덤 요소(의외성)가 최소 1개 이상 존재함까?
- [ ] 타격음+이펙트가 충분히 "꽂히는 느낌"을 주슴까?
- [ ] 같은 전투 100번 반복해도 매번 다른 경험이 될 구조임까?

**[BM 연결 검증]**
- [ ] "이 캐릭터/장비가 전투에서 확실히 다르다"는 체감이 설계되었슴까?
- [ ] 과금 없이도 전략으로 극복 가능한 구간이 있슴까? (P2W 방지)
- [ ] 전투 막힘 → 과금 유인의 자연스러운 흐름이 설계되었슴까?
- [ ] 오토 전투 관련 편의 과금이 설계되었슴까?

**[수명 검증]**
- [ ] 전투 메커닉 변경 없이 1년 이상 신선함을 유지할 방법이 있슴까?
- [ ] PvP 또는 협동 소셜 레이어가 설계되었슴까?
- [ ] 장기 유저의 투자 가치(육성한 캐릭터의 영구 가치)가 보존됨까?
