## 원칙 0. 성장 시스템의 목적

> **"다음 날 또 켜게 만드는 것"**

기획 철학서의 재미 기준과 동일하다. 성장 시스템은 그 자체가 목적이 아니라, **"한 세션이 끝났을 때 유저에게 아직 해결되지 않은 미완의 목표가 최소 1개 남아있는가?"**를 달성하기 위한 수단이다.

성장 시스템이 잘 설계된 게임은:
- 오늘 "조금 더 강해졌다"를 느끼게 하고
- 내일 "더 강해지면 열리는 것"을 보여준다

## 원칙 1. 성장 레이어는 5~7층을 지켜라

### 데이터 근거

| 레이어 수 | 대표 결과 | 판정 |
|-----------|-----------|------|
| 2~3층 | Clash Royale $3.3B, Brotato 인디 히트 | 성공 가능하나 깊이 부족. 극도로 중독적인 핵심 루프가 있을 때만 유효 |
| 4~5층 | Arknights $1.1B, NIKKE $1B, Legend of Mushroom $270M(6개월) | 성공. 직관성과 깊이의 최적 밸런스 |
| 5~7층 | 원신 $50억+, 스타레일 $20억+, AFK Arena $14.5억 | 성공. 모바일 RPG의 황금 구간 |
| 8~10층 | 에픽세븐, RAID, 킹스레이드 | 코어 유저에겐 깊이 있으나 신규 진입장벽 높음 |
| 10층+ | 검은사막M, 로스트아크(삭제 결정) | 반드시 후회. 로스트아크가 초월/엘릭서 삭제한 것이 증거 |

### 적용 방법

1. 최초 설계 시 **핵심 5층을 먼저 확정**한다
2. 라이브 운영에서 레이어를 추가할 때마다 **"이 레이어는 향후 삭제될 수 있는가?"를 사전 검토**한다
3. 8층을 넘기면 경고, 10층을 넘기면 반드시 정리 계획을 세운다

### 예외

- 게임의 핵심 게임성 자체가 "복잡한 빌딩"인 경우 (예: PoE 류) 레이어가 더 많아도 된다
- 단, 우리 팀의 유니크 전략이 "복잡한 빌딩"이 아닌 이상, 이 예외는 적용하지 않는다

## 원칙 2. 실패에도 보상이 있어야 한다 (Anti-Failure)

### 데이터 근거

| 방식 | 대표 | 효과 |
|------|------|------|
| 가챠 천장(하드) | 원신(90연차 5성 확정) | 무과금 계획적 과금 가능. $50억+ |
| 실패 누적 보상 | 로스트아크 아티산 에너지 | 실패해도 게이지 축적 → 확정 성공. 업계 표준화 |
| 런 실패 보상 | Hades, Archero | 실패 런에서도 재화/해금. 이탈 방지 핵심 |
| 보장형 강화 | 천애명월도M | 실패해도 일정 성장 보장. 유저 스트레스 최소 |
| 파괴형 강화 (反면교사) | 리니지2M | 장비 파괴로 수개월 투자 증발. 단기 과금 극대화되나 장기 생태계 파괴 |

### 적용 방법

어떤 형태의 성장 시스템이든, **유저의 투자(시간/재화/과금)가 "완전히 0"이 되는 경우가 없어야 한다.**

```
[체크리스트]
□ 강화 실패 시: 재료만 소모되는가? 아니면 기존 수치도 하락/파괴되는가?
  → 하락/파괴는 2024년 이후 시장에서 퇴출 추세. 채택하지 않는다
□ 가챠 실패 시: 천장 시스템이 있는가?
  → 하드 천장(확정) 필수. 소프트 천장(확률 보정)만으로는 부족
□ 런 실패 시: 아무것도 얻지 못하는 런이 존재하는가?
  → "제로 진행 런"은 제거한다. 최소한의 재화/경험/해금 진행이 있어야 한다
□ 강화 확률이 존재한다면: 실패 누적 보상(아티산 에너지 등)이 있는가?
  → 없으면 추가한다
```

## 원칙 3. 성장은 "숫자 올리기"가 아니라 "선택지 확대"를 지향하라

### 데이터 근거

$1B+ 매출 클럽의 공통 특징:

| 게임 | 매출 | 성장 = 선택지 확대 방식 |
|------|------|----------------------|
| FEH | $1B+ | 스킬 계승으로 "같은 캐릭터, 다른 빌드" |
| Arknights | $1.1B+ | 승진 → 새 스킬 특성 해금 → 새로운 전략 가능 |
| NIKKE | $1B+ | 오버로드 장비 옵션으로 빌드 다양화 |
| Rise of Kingdoms | $3.5B+ | 탈랜트 트리 분기 선택 → 플레이스타일 결정 |
| Hades | 인디 최대급 | Mirror of Night로 플레이스타일 커스터마이징 |

반면, 순수 파워 인플레이션만 있는 게임:
- 검은사막M: "강해져도 즐길 컨텐츠 부족" → 유저 이탈 가속
- 리니지2M: 수치만 오르는 성장 → "강화의 목적성" 상실

### 적용 방법

성장 레이어를 설계할 때 각 레이어가 **유저에게 새로운 "선택"을 주는지** 확인한다.

```
[좋은 예]
레벨 30 달성 → 2번째 스킬 슬롯 해금 → "어떤 스킬을 장착할까?" (선택)
장비 티어 업 → 새로운 세트 효과 선택지 → "어떤 세트로 갈까?" (선택)
돌파 완료 → 새로운 모드/스테이지 접근 → "다음엔 무엇을 도전할까?" (선택)

[나쁜 예]
레벨 30 → 공격력 +50 (선택 없음, 숫자만 증가)
장비 강화 → 방어력 +100 (선택 없음, 숫자만 증가)
```

모든 성장 레이어에 선택지를 넣을 필요는 없다. 하지만 **전체 레이어의 최소 절반 이상은 유저에게 의미 있는 선택을 요구해야 한다.**

## 원칙 4. 성장 레이어는 순차 해금하라

### 데이터 근거

- AFK Arena: 어센션 → 공명수정 → 시그니처 아이템 → 가구 → 조각상 순차 해금. 교과서적 성공 ($14.5억)
- WotV:FFBE: 7층 이상 레이어를 초반부터 노출 → 신규 유저 "뭐가 이렇게 많아?" → 이탈
- 원신/스타레일: 모험 등급에 따라 시스템 순차 해금 → $50억+ / $20억+

### 적용 방법

```
[성장 레이어 해금 템플릿]

[초반 - 0~2시간]
해금되는 것: 레이어 1(기본 레벨링), 레이어 2(장비 장착)
숨기는 것: 레이어 3~7 (존재조차 보이지 않게)
목적: "이 게임 어떻게 하는 거구나"를 이해시킨다

[중초반 - 2~10시간]
해금되는 것: 레이어 3(스킬/돌파), 레이어 4(세트 장비)
목적: "더 강해지려면 이것도 해야 하는구나" → 깊이를 인식

[중반 - 10~30시간]
해금되는 것: 레이어 5(고급 커스터마이징/빌드), 레이어 6(엔드게임 파밍)
목적: "내 캐릭터를 내가 원하는 방향으로 키운다" → 소유감

[후반/엔드게임 - 30시간+]
해금되는 것: 레이어 7(최적화/미세 조정)
목적: "완벽한 빌드를 향한 무한 추구" → 장기 리텐션
```

**핵심**: 유저가 "뭘 해야 할지 모르겠다"고 느끼면 그 시점에서 해금된 레이어가 너무 많은 것이다.

## 원칙 5. 엔드게임에는 "랜덤 장비 파밍"이 있어야 한다

### 데이터 근거

2014년 서머너즈 워 룬 시스템 이후 10년간, 장르를 막론하고 모든 성공한 RPG의 엔드게임은 **랜덤 옵션 장비 파밍**이다.

```
[10년 불변의 계보]
서머너즈 워 룬(2014) → 에픽세븐 장비(2018) → 원신 성유물(2020)
  → 스타레일 유물(2023) → ZZZ 디스크(2024) → 명조 에코(2024)
```

### 왜 이 구조가 10년간 살아남았는가

1. **무한한 목표**: "완벽한 옵션"은 이론상 달성 가능하나 확률적으로 극히 어려움 → 영원히 추구 가능
2. **간헐적 강화 스케줄**: 좋은 옵션이 나왔을 때의 쾌감이 도파민 분비 → 슬롯머신과 동일 심리
3. **자연스러운 과금 연계**: 스태미나 구매, 즉시 파밍으로 "더 많이 돌리고 싶다" 유도
4. **커뮤니티 활성화**: "내 장비 스펙" 공유 → 소셜 참여

### 적용 방법

우리 게임의 유니크한 핵심 게임성이 무엇이든, 엔드게임에는 **핵심 게임성과 연결된 랜덤 장비 파밍** 구조를 설계한다.

단, 이것은 유니크 영역이 아니라 **참고 영역**이다. 기획 철학서의 원칙에 따라 검증된 레퍼런스(서머너즈 워~ZZZ 계보)를 충실히 참고한다.

```
[설계 시 체크리스트]
□ 장비에 주옵션 + 부옵션(랜덤)이 있는가?
□ 세트 효과가 있어 조합의 재미를 주는가?
□ 파밍 자체가 "하기 싫은 노동"이 아닌 "재미있는 콘텐츠"인가?
  → 스타레일의 모의우주가 교과서. 파밍 던전 자체가 로그라이크 콘텐츠
□ 부옵션의 랜덤 범위가 "의미 없이 넓지" 않은가?
  → 너무 넓으면 좌절감. 적절한 범위에서 "좋은 것 vs 완벽한 것"의 차이로 설계
```

## 원칙 6. 세션 길이에 맞는 성장 속도를 설계하라

### 데이터 근거

| 타겟 세션 | 성장 피드백 속도 | 최적 레이어 | 성공 사례 |
|-----------|----------------|------------|-----------|
| 3~5분 | 매 세션 즉시 보상 | 2~3층 | Clash Royale, Brawl Stars |
| 5~10분 | 2~3세션마다 레벨업/장비 | 3~4층 | Archero, FEH, Arknights |
| 10~15분 | 세션 내 성장 아크 완성 | 3~5층 | Survivor.io, Soul Knight |
| 15~30분 | 다중 보상 + 미완 목표 | 5~7층 | 원신, 스타레일, NIKKE |
| 30분+ | 장기 파밍 + 깊은 성장 | 7층+ | MMO, WotV, Diablo Immortal |

### 적용 방법

**먼저 타겟 세션 길이를 확정하고, 그에 맞는 성장 속도를 역산한다.**

```
[공식]
타겟 세션 = 10분이라면:
  → 10분 안에 최소 1번의 "강해졌다" 느낌을 줘야 한다
  → 10분 안에 최소 1개의 "다음에 해야 할 것"이 보여야 한다
  → 성장 레이어는 3~5층으로 제한한다
  → 각 레이어의 한 단계 소요 시간 = 3~5세션(30~50분) 이내
```

**절대 법칙**: 성취감의 주기가 1주일을 넘으면 안 된다 (기획 철학서 3-3 참조). 어떤 세션 길이든 주 1회 이상의 의미 있는 성장을 보장해야 한다.

## 원칙 7. 단순함이 복잡함을 이길 수 있다

### 데이터 근거

| 게임 | 성장 레이어 | 매출 | 핵심 루프 |
|------|-----------|------|-----------|
| Clash Royale | 2층 | $3.3B+ | 카드 레벨업 하나. 마스터리는 메타 이해에서 |
| Legend of Mushroom | 4층 | $270M (6개월) | 매직 램프→장비→판매 단일 루프 |
| Capybara Go | 5층 | $100M (3개월) | 텍스트 로그라이크+방치 단일 루프 |
| Vampire Survivors | 2층 | 인디 최대급 | 이동만. 미세한 메타 업그레이드 |

### 적용 방법

성장 시스템을 설계할 때, **"이 게임의 핵심 루프가 그 자체로 중독적인가?"**를 먼저 판단한다.

- **핵심 루프가 극도로 중독적이라면** → 성장 시스템은 얇게 가도 된다 (2~4층)
- **핵심 루프가 보통 수준이라면** → 성장 시스템의 깊이로 리텐션을 보완해야 한다 (5~7층)

이것은 기획 철학서의 **"유니크한 핵심에 올인하고, 나머지는 검증된 것으로"** 원칙과 동일한 맥락이다.

**경고**: "복잡하면 깊이가 있어 보인다"는 착각이다. 레이어를 추가하기 전에 **"이 레이어 없이도 유저가 다음 날 돌아오는가?"**를 먼저 물어라. 돌아온다면 그 레이어는 불필요하다.

## 원칙 8. 신규/복귀 유저 따라잡기 시스템을 1일차부터 고려하라

### 데이터 근거

- NIKKE 싱크로 디바이스: 상위 5체 레벨 = 전체 레벨. 신캐릭터 즉시 전투 합류 가능 → $1B+
- AFK Arena 공명수정: 핵심 5인 레벨업 → 나머지 전체 동기화
- 검은사막M: 따라잡기 없음 → 신규 유저 "3년 뒤에야 경쟁 가능" → 유입 차단

### 적용 방법

```
[따라잡기 시스템 설계 체크리스트]
□ 신규 유저가 "현재 코어 유저의 70% 전투력"에 도달하는 시간은?
  → 2주 이내가 이상적. 1개월 이상이면 이탈 위험
□ 새 캐릭터를 뽑았을 때 즉시 실전 투입 가능한가?
  → 동기화 시스템(NIKKE/AFK Arena) 참고
□ 라이브 운영에서 성장 레이어가 추가될 때 복귀 유저의 부담은?
  → "따라잡기 서버", "누적 보상 패키지" 등 사전 설계
□ 코어 유저와 신규 유저가 함께 즐길 수 있는 콘텐츠가 있는가?
  → 협동 콘텐츠에서 레벨 차이가 크면 함께 플레이 불가 → 소셜 요소 무력화
```

## 원칙 9. 과금은 "성장 속도"를 팔고, "성장 자체"를 팔지 마라

### 데이터 근거

| 과금 방식 | 대표 | 유저 반응 | 장기 결과 |
|-----------|------|-----------|-----------|
| 시간 단축 판매 | AFK Arena | "과금 안 해도 언젠가는 도달" → 수용 | $14.5억. 장기 운영 |
| 직접 파워 판매 | 검은사막M, 리니지2M | "돈 안 쓰면 게임 못 함" → 분노 | 단기 고매출 후 감소 |
| 옵션/외형 판매 | 알비온, 블소 NEO | "게임 자체는 공정" → 호의적 | 장기 유지. 코어 커뮤니티 형성 |
| 편의 판매 | 원신/스타레일 | "레진/체력 리필" → 적절한 수용 | $50억+ / $20억+ |

### 적용 방법

기획 철학서 BM 철학(7장)과 연동:

1. **"갖고 싶은 것"을 먼저 만든다** → 성장 시스템 안에서 가치 있는 목표를 설정
2. **그것에 도달하는 여정을 설계한다** → 무과금도 시간 투자로 반드시 도달 가능하게
3. **여정을 "빠르게" 하는 것을 판매한다** → 도달 불가능한 것을 판매하는 것이 아님

**금지 사항**:
- 과금 없이는 달성 불가능한 성장 단계를 만들지 않는다
- 과금 유저와 무과금 유저의 "도달 가능한 최종 목표"는 동일해야 한다
- 차이는 "속도"뿐이어야 한다

## 원칙 10. 파밍 콘텐츠 자체가 재미있어야 한다

### 데이터 근거

| 게임 | 파밍 콘텐츠 | 유저 반응 |
|------|------------|-----------|
| 스타레일 모의우주 | 유물 파밍 던전이 로그라이크 미니게임 | "파밍이 재미있다" — 장르 최초 수준의 호평 |
| 명조 에코 사냥 | 오픈월드에서 적 처치 = 장비 획득 | "파밍이 탐험과 연결되어 자연스럽다" |
| 에픽세븐 | 반복 던전 오토 | "노동이다" — 장비 RNG와 결합하여 최악의 경험 |
| 서머너즈 워 카이로스 | 반복 던전 오토 | "단순 반복" — 그러나 10년 운영에 성공 |

### 적용 방법

기획 철학서의 **"재미의 공식(성취감 x 의외성)"**을 파밍 콘텐츠에도 적용한다.

```
[파밍 콘텐츠 재미 체크]
□ 파밍 과정에서 의외성이 있는가? (매번 같은 경험이 아닌가?)
  → 스타레일 모의우주: 매 런마다 다른 축복 조합
□ 파밍 결과에서 성취감이 있는가?
  → 좋은 옵션 장비가 나왔을 때의 쾌감
□ 파밍에 "선택"이 존재하는가?
  → 어떤 던전을 파밍할지, 어떤 세트를 노릴지
□ 오토 파밍이 가능하되, 수동 파밍이 더 효율적인 구조인가?
  → 캐주얼 유저(오토)와 코어 유저(수동)를 동시에 만족
```

## 원칙 적용 순서

성장 시스템을 설계할 때 이 원칙들을 아래 순서로 적용한다.

```
Step 1: 게임의 타겟 세션 길이를 확정한다 (원칙 6)
        ↓
Step 2: 세션 길이에 맞는 성장 레이어 수를 결정한다 (원칙 1)
        ↓
Step 3: 핵심 루프의 중독성을 평가한다 (원칙 7)
        → 핵심 루프가 충분히 중독적이면 레이어를 줄인다
        → 부족하면 레이어의 깊이로 보완한다
        ↓
Step 4: 각 레이어가 "선택지를 확대"하는지 확인한다 (원칙 3)
        ↓
Step 5: 레이어의 순차 해금 순서를 설계한다 (원칙 4)
        ↓
Step 6: 엔드게임 파밍 루프를 설계한다 (원칙 5, 원칙 10)
        ↓
Step 7: 실패 보상 메커니즘을 모든 확률 요소에 적용한다 (원칙 2)
        ↓
Step 8: 과금 포인트를 "속도 판매"로 설계한다 (원칙 9)
        ↓
Step 9: 신규/복귀 유저 따라잡기 시스템을 설계한다 (원칙 8)
        ↓
Step 10: 땡초김밥에게 검증 요청
```

---

## 검증 시 이 원칙의 사용법 (땡초김밥용)

성장 시스템 기획안을 검증할 때, 다음을 체크한다:

```
[성장 시스템 검증 체크리스트]

□ 원칙 1: 성장 레이어가 5~7층 범위인가?
□ 원칙 2: 모든 확률 요소에 실패 보상이 있는가?
□ 원칙 3: 최소 절반 이상의 레이어에서 유저에게 선택을 요구하는가?
□ 원칙 4: 레이어가 순차 해금되는가? 초반에 전부 노출되지 않는가?
□ 원칙 5: 엔드게임 파밍 루프가 존재하는가?
□ 원칙 6: 타겟 세션 길이와 성장 속도가 맞는가?
□ 원칙 7: 핵심 루프의 중독성 대비 레이어 깊이가 적절한가?
□ 원칙 8: 신규/복귀 유저 따라잡기 시스템이 있는가?
□ 원칙 9: 과금이 "속도 판매"인가? "성장 자체 판매"는 아닌가?
□ 원칙 10: 파밍 콘텐츠 자체에 재미(성취감 x 의외성)가 있는가?
```
