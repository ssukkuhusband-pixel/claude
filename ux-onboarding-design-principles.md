## 원칙 0. UX의 본질 — "참신한 것을 즉시 이해시키는 것"

> 아무리 유니크하고 재미있어도 유저가 이해 못하면 이탈한다.

20종 분석에서 가장 명확한 패턴: **D1 리텐션이 높은 게임일수록 첫 조작까지의 시간이 짧다.**

| 첫 조작 시간 | 대표 게임 | D1 리텐션 |
|------------|---------|----------|
| 2~5초 | Vampire Survivors, Candy Crush, Survivor.io | 38~65% |
| 5~10초 | Archero, Balatro, Slay the Spire | 35~70% |
| 30~90초 | Genshin, NIKKE, HSR | 30~40% |

운수좋은날의 핵심 과제: **"슬롯머신으로 싸운다"는 참신한 메커닉을 5초 안에 체험시키는 것.**

## 원칙 1. 첫 5분의 법칙 — D1 리텐션은 여기서 결정

### 10초-30초-5분 법칙

```
[10초] 유저가 "이 게임이 뭘 하는 건지" 이해
  → 이해 못하면 그냥 닫음

[30~45초] 첫 행동 발생
  → 슈터면 총을 쏨, 슬롯이면 릴이 돌아감, 퍼즐이면 첫 매치

[5분] "또 하고 싶다" 상태 도달
  → 최소 1개의 미완의 목표가 남아있어야 D1 리텐션 발생
```

### Aha Moment 3단계

```
① Micro-Win (0~30초) → "내가 잘했다" 느낌
② Core Fun 체험 (1~3분) → "이 게임만의 재미" 각인
③ 욕심 유발 (3~5분) → "더 하고 싶다" 상태
```

### 첫 세션에서 반드시 전달해야 하는 3가지

| 순서 | 전달 내용 | 방법 |
|------|---------|------|
| ① | 핵심 재미가 뭔지 | **직접 체험** (설명 금지) |
| ② | 내가 성장하고 있다는 느낌 | 숫자/레벨업/보상 시각 피드백 |
| ③ | 다음에 뭘 해야 하는지 | 미완의 목표, 잠긴 기능 미리보기 |

### 데이터

- 모바일 신규 유저 첫 세션 평균: **8분** — 이 안에 "또 하고 싶다"를 심어야 함
- 첫 3~5분이 가장 많은 이탈 구간
- 튜토리얼 최적화 후 D1 리텐션 **+18%** 달성 사례 (sonamine.com)

### 체크리스트

```
□ 10초 안에 "이 게임이 뭔지" 전달되는가?
□ 30초 안에 첫 행동이 발생하는가?
□ 5분 안에 "또 하고 싶다" 상태가 되는가?
□ Aha Moment 3단계가 설계되어 있는가?
□ 미완의 목표가 최소 1개 남는가?
```

## 원칙 2. 튜토리얼 — "가르치지 말고 체험하게 하라"

### 데이터

| 방식 | 완료율 |
|------|--------|
| 3단계 가이드 | **72%** |
| 7단계 가이드 | **16%** |

### 원칙

- **인터랙티브 학습 > 텍스트 설명** — Apple 공식 권고
- **하이브리드가 정답**: 핵심 2~3개만 강제 교육, 나머지 자연 탐색 (Clash Royale식)
- **텍스트 팝업 금지**: 숫자와 시각 이펙트로 결과를 보여줘야 함
- 강제 튜토리얼은 **5분 이내** 완료되어야 함

### 참신한 메커닉 교육 3법칙

이것이 운수좋은날에 가장 중요한 원칙임다:

**법칙 1: 제일 쉬운 버전을 먼저**
- 첫 런은 결과가 무조건 좋게 나오도록 설계
- "이렇게 하면 강하다"를 먼저 각인
- Balatro: 첫 Ante 목표(300칩)를 극히 낮게 설정 → 반드시 성공

**법칙 2: 텍스트 설명 금지, 숫자와 시각으로**
- 인지 부하 이론: 작업 기억 = 7±2개 항목 한계
- 새 메커닉 설명에 글자가 들어가면 즉시 인지 한계 초과
- Balatro: 카드 선택하면 즉시 "Two Pair" 표시 → 족보 암기 불필요

**법칙 3: 새 메커닉 = 즉각적 성취감 연결**
- 첫 체험에서 성취감(숫자 폭발, 클리어, 리워드)이 즉시 따라와야 함
- Balatro: 첫 조커 획득 → 점수 5~10배 급등 → 텍스트 없이 "조커가 핵심"을 숫자로 이해

### 운수좋은날 적용

```
[나쁜 방법] 팝업: "5×3 슬롯에서 같은 심볼이 3개 이상 연결되면 매치 성공!"
[좋은 방법] 첫 전투 → 슬롯 자동 스핀 → 화염 3매치 → 화면에 "🔥×3 = 150 데미지!" 팡 → 적 HP 반토막
```

### 체크리스트

```
□ 강제 튜토리얼이 5분 이내인가?
□ 텍스트 설명 대신 체험으로 가르치는가?
□ 첫 전투가 반드시 승리하게 설계되어 있는가?
□ 핵심 메커닉의 결과가 숫자/이펙트로 즉시 보이는가?
□ 특수 요소(스킬/심볼)가 2~3분 안에 처음 등장하는가?
```

## 원칙 3. Thumb Zone — 한 손 조작의 물리 법칙

### 데이터

- 스마트폰 터치의 **75%가 엄지**로 이루어짐
- 최소 탭 타깃: **48×48px** (Google Material Design)

### 화면 3구역

```
┌─────────────────┐
│  🔴 RED ZONE    │ ← 양손 필요 (중요 기능 배치 금지)
│  (위쪽 코너)     │
├─────────────────┤
│  🟡 YELLOW ZONE │ ← 뻗어야 닿음 (2순위 기능)
│  (중간 영역)     │
├─────────────────┤
│  🟢 GREEN ZONE  │ ← 엄지가 자연히 쉬는 곳
│  (화면 하단)     │    핵심 버튼, 네비게이션 배치
└─────────────────┘
```

### 원칙

- **가장 자주 하는 행동 = 가장 적은 터치 = GREEN ZONE**
- 런 시작: 메인 화면 중앙 큰 버튼 **1탭**
- 스킬 선택: 카드 3개 **화면 하단**, 1탭 선택
- 런 재시작: 사망 화면 하단 "다시 하기" **1탭**

### 체크리스트

```
□ 핵심 플레이 버튼이 GREEN ZONE(하단)에 있는가?
□ 가장 자주 하는 행동이 2탭 이내인가?
□ 탭 타깃이 48×48px 이상인가?
□ 한 손으로 모든 핵심 조작이 가능한가?
```

## 원칙 4. 정보 과부하 방지 — Miller's Law 적용

### 데이터

- Miller's Law: 작업 기억 = 동시 **7±2개** 항목 처리 한계
- 실전 HUD 권고: 핵심 정보 **3~5개** 이하

### 정보 계층 (Progressive Disclosure)

```
1순위 (항상 표시): HP, 점수/타이머, 핵심 조작 버튼
2순위 (필요시): 스킬 상태, 미니맵, 퀘스트 방향
3순위 (탭해서): 상세 스탯, 인벤토리, 설정
```

### 순차적 시스템 해금

- Day 1: 핵심 전투만
- Day 2~3: PvP/소셜 해금
- Day 4~5: 길드/클랜
- Day 7+: 심층 콘텐츠

**로그라이트 적용**: 첫 런 스킬 선택지 **5~8개**로 제한 → 점진적으로 해금. 선택지 과다 = 초기 이탈 원인.

### 3-레이어 목표 시스템

```
즉각 (0~10분): 화면 화살표, 강조 — "지금 이걸 해"
단기 (오늘): 로비에 항상 표시 — 일일 퀘스트, 보상
장기 (이번 주): 탭 안에 배치 — 성장/수집 목표
```

### 체크리스트

```
□ HUD 핵심 정보가 5개 이하인가?
□ 시스템이 순차적으로 해금되는가? (Day 1에 전부 노출 금지)
□ 3-레이어 목표(즉각/단기/장기)가 설계되어 있는가?
□ "뭘 해야 할지 모르겠다" 상황이 발생하지 않는가?
```

## 원칙 5. 알림/배지 — Red Dot Fatigue 방지

### 데이터

- 배지 있는 앱: 오픈율 **+16~25%**
- 과다 시: "Red Dot Blindness" — 뱃지 너무 많으면 아무것도 안 보임
- 저강도 불안감 → 장기적 앱 사용량 감소

### 원칙

| 배지 OK | 배지 금지 |
|---------|----------|
| 일일 보상 수령 가능 | 무의미한 "돌아오세요" |
| 새 콘텐츠 해금 (1회만) | 이미 아는 정보에 반복 |
| 이벤트 마감 임박 | 3개+ 탭 동시 배지 |

### 체크리스트

```
□ 배지가 동시에 3개 이상의 탭에 표시되지 않는가?
□ 배지가 한 번 확인 후 사라지는가?
□ "청소 플레이"(배지 없애기 위한 탭)를 유발하지 않는가?
```

## 원칙 6. 과금 노출 타이밍 — 너무 빠르면 이탈

### 데이터

- 리워드 광고를 FTUE에서 체험한 유저 → IAP 전환 **5배** 높음
- "유저가 게임에 투자됐다고 느낄 때"가 최적 타이밍
- 첫 과금 늦출수록 LTV 높아짐 (solar-engine.com)

### 20종 분석 결과

| 과금 노출 타이밍 | 대표 게임 | D1 리텐션 |
|----------------|---------|----------|
| 3분 이내 | Coin Master | 30~35% (낮음) |
| 5~7분 | Archero, Survivor.io, AFK Arena | 35~53% |
| 10~15분 | Brawl Stars, Clash Royale, CoC | 40~50% |
| 30분+ | Genshin Impact | 35~40% |

### 최적 타이밍 공식

```
너무 빠름 (첫 세션 내 즉시) → 신뢰 붕괴 → 이탈
적당함 (코어 루프 이해 후, 첫 벽) → 자연스러운 전환
너무 늦음 → 기회 상실

운수좋은날 권장: 첫 5분 순수 게임 → 7분 이후 첫 상점 노출
```

### 체크리스트

```
□ 첫 5분 동안 과금 요소가 노출되지 않는가?
□ 첫 과금 노출이 "행동 트리거"(실패/자원 부족) 기반인가?
□ 리워드 광고가 FTUE 중에 자연스럽게 1회 체험되는가?
```

## 원칙 7. 로그라이트 특화 UX — 런 흐름과 사망 화면

### 런 시작 흐름

```
로비 → [런 시작 1탭] → (캐릭터 선택 시 2탭) → 즉시 게임
총 2탭 이내. 확인 팝업/로딩 설명 금지.
```

### 스킬 선택 UI (3개 중 1택)

```
┌──────────────────────────────────┐
│  [스킬 A]  [스킬 B]  [스킬 C]    │
│   아이콘    아이콘    아이콘       │
│   이름      이름      이름        │
│   설명(1줄) 설명(1줄) 설명(1줄)   │
│                                   │
│ (하단 배치 — GREEN ZONE)          │
└──────────────────────────────────┘
```

- 카드 3개 = 기억 가능한 최대. 4개부터 비교 피로
- 설명 **1~2줄** 이내
- 선택 후 **즉각 시각/음향 피드백**
- 화면 **하단~중앙** 배치 (한 손 조작 가능)

### 빌드 정보 표시

- 현재 보유 스킬 = 화면 가장자리 작은 아이콘 (플레이 방해 금지)
- 상세 정보 = 탭해서 확인
- 경험치/레벨 = 눈에 띄되 최소 크기

### 사망 화면 — "한 판만 더" 유도

이것이 로그라이트 UX의 **가장 중요한 화면**임다.

```
┌──────────────────────────────────────┐
│  [이번 런 결과]                       │
│  시간: X분 X초   처치: N명            │
│  최고 딜: X,XXX   도달 스테이지: N    │
│                                       │
│  [이번 런으로 획득한 영구 보상]        │
│  "장비 조각 +15, 골드 +2,340"         │
│                                       │
│  [다음 런에서 해금될 것]              │  ← 핵심!
│  "레벨 N 달성 시 [A 스킬] 해금"       │
│                                       │
│  ████ [다시 하기] ████   [로비로]     │  ← "다시 하기"가 더 크고 가까이
└──────────────────────────────────────┘
```

**사망 화면 5대 원칙:**

1. **결과는 간결하게** — 숫자 3~4개. 과다 통계 = 이탈
2. **다음 런의 이유** — "다음엔 이 스킬 나올 수 있어"
3. **영구 보상 표시** — "이번 런도 헛되지 않았다"
4. **"다시 하기" = 주 CTA** — 로비 버튼보다 크고, 가까이
5. **감각적 카타르시스** — 사망 순간 이펙트/음악으로 정서적 흥분 유지 → 흥분 상태에서 재시작

### 체크리스트

```
□ 런 시작이 2탭 이내인가?
□ 스킬 선택 카드가 GREEN ZONE에 배치되어 있는가?
□ 사망 화면에 "다음 런의 이유"가 표시되는가?
□ "다시 하기" 버튼이 "로비" 버튼보다 크고 가까운가?
□ 사망 화면에 영구 보상 획득이 표시되는가?
```

## 원칙 8. 복귀 유저 UX — 리온보딩

### 원칙

- **앱 열자마자**: 환영 보상 + 변경 요약 + 빠른 재진입 버튼
- **UI 대규모 변경 금지** — 복귀 유저 혼란의 최대 원인
- **튜토리얼 재표시 = 모욕** — 반드시 스킵 또는 없애야 함
- 새 기능은 강조 표시 1회 후 사라지게

### 복귀 보상 스케일

| 미복귀 기간 | 보상 규모 |
|-----------|---------|
| 7일 | 소량 자원 + 새 기능 미리보기 |
| 30일 | 중간 규모 복귀 패키지 |
| 90일+ | 대형 환영 복귀 이벤트 |

### 체크리스트

```
□ 복귀 유저에게 "뭐가 바뀌었는지" 한 화면 요약이 있는가?
□ 튜토리얼 재강제가 없는가?
□ 복귀 보상이 미복귀 기간에 비례하는가?
```

## 원칙 9. UI 일관성과 피드백

### 모바일 게임 UI 실수 TOP 5

| # | 실수 | 결과 |
|---|------|------|
| 1 | 불명확한 온보딩 | 첫 3~5분 이탈 (최다 원인) |
| 2 | 정보 과부하 | 결정 피로 → 이탈 |
| 3 | 탭 피드백 없음 | "눌렀나?" 혼란 |
| 4 | 48px 미만 탭 타깃 | 오조작 → 짜증 |
| 5 | 불일관한 디자인 언어 | 학습 비용 계속 상승 |

### 피드백 원칙

- 모든 탭에 **시각+음향 피드백** 필수
- 성공 시: 숫자 팡, 이펙트, 효과음
- 실패 시: 화면 흔들림, 짧은 진동
- 스킬 선택 시: 카드 확대 + 획득 이펙트

### 메인 화면(로비) UI 공통점 (성공 게임 분석)

| 패턴 | 대표 |
|------|------|
| 하단 탭 네비게이션 | Clash Royale, CoC |
| 핵심 플레이 버튼 화면 중앙 | 거의 모든 로그라이트 |
| BM/상점은 탭 안에 분리 | 성공 게임 공통 |
| 이벤트 = 배지로, 팝업 최소화 | Genshin Impact |

### 체크리스트

```
□ 모든 탭에 시각+음향 피드백이 있는가?
□ 디자인 언어(아이콘, 컬러, 폰트)가 전체 화면에서 일관적인가?
□ 메인 화면에 핵심 플레이 버튼이 가장 눈에 띄는가?
□ BM/상점이 메인 플레이를 방해하지 않는가?
```
