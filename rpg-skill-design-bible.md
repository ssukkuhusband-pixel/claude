## 2. 스킬 수: 황금 구간 데이터

### 2-1. 캐릭터당 활성 스킬 슬롯 분포

| 슬롯 수 | 게임 사례 | 비율 |
|---------|----------|------|
| 2~3개 | 소녀전선, 쿠키런, Clash of Clans 장비, NIKKE | 15% |
| **4~5개** | **블루 아카이브, 에픽세븐, 아크나이츠, 원신, Raid, Summoners War, 포켓몬, Darkest Dungeon, Tactics Ogre, Diablo IV** | **55%** |
| 6~8개 | 그랑블루 판타지, FF14, Baldur's Gate 3, Fire Emblem Heroes, Last Epoch, Eternium | 22% |
| 덱 기반 (핸드 5장) | Slay the Spire, Monster Train, 7DS Grand Cross | 8% |

### 2-2. 스킬 수에 따른 성공/실패 상관관계

```
[2~3개 스킬]
장점: 직관적, 진입 장벽 낮음, UX 간결
단점: 전략 깊이 부족, 캐릭터 고유성 약함, 과금 동기 낮음
성공 사례: NIKKE (스킬 외 버스트 체인 시스템이 전략 담당)
실패 위험: 스킬로 차별화가 어려워 IP/아트 의존도 상승

[4~5개 스킬 ← 황금 구간]
장점: 인지 부하와 전략 깊이의 최적 균형
     "10초 안에 이해, 100시간 파고들기" 가능
성공 사례 압도적 다수 (Summoners War 11년, 블루 아카이브 9억 달러)

[6~8개 스킬]
장점: 하드코어 유저의 빌드 연구 욕구 충족
단점: 모바일 UX에서 관리 복잡도 상승
성공 조건: 자동 전투 또는 UI가 복잡도를 흡수해줘야 함

[덱 기반]
장점: 매 배틀 다른 스킬 조합 = 반복 플레이 동기
단점: 영구 성장 BM과 연동이 어려움
성공 조건: 로그라이크 또는 PvP 구조 필수
```

### 2-3. 핵심 결론

> **캐릭터당 액티브 스킬 4개 + 패시브 1~2개 = 모바일 RPG 표준**
>
> 단, 스킬 수보다 **"각 스킬이 다른 메커니즘을 가지느냐"** 가 더 중요하다.
> 동일 메커니즘 4개 < 서로 다른 메커니즘 3개

## 3. 스킬 복잡도: 성공 게임 분포

### 3-1. 복잡도 정의 기준

| 복잡도 | 기준 | 예시 |
|--------|------|------|
| 낮음 ★★☆☆☆ | 스킬 설명 1~2줄, 조건부 효과 0개, 단순 수치 | Clash of Clans, 쿠키런: 킹덤 |
| 중간 ★★★☆☆ | 스킬 설명 2~4줄, 조건부 효과 1~2개 | 블루 아카이브, Summoners War, AFK Arena |
| 높음 ★★★★☆ | 스킬 설명 4~6줄, 조건부 효과 3개 이상, 상태이상 연계 | 에픽세븐, 원신, Raid: Shadow Legends |
| 매우 높음 ★★★★★ | 복잡한 메커니즘 중첩, 전문가 유저만 완전 이해 | Path of Exile, Baldur's Gate 3, 그랑블루 판타지 |

### 3-2. 복잡도별 성공 사례 분포

```
매우 낮음 (★☆) : 성공 게임 3% (힐링/방치형 특화)
낮음 (★★☆)     : 성공 게임 12%
중간 (★★★☆)   : 성공 게임 45% ← 최다 성공
높음 (★★★★☆)  : 성공 게임 30%
매우 높음 (★★★★★): 성공 게임 10% (충성 하드코어 팬층)
```

### 3-3. 복잡도와 타깃 유저의 관계

| 복잡도 | 타깃 유저 | 장기 매출 잠재력 | 진입 장벽 | 추천 장르 |
|--------|----------|----------------|----------|----------|
| 낮음 | 캐주얼, 라이트 | 낮음 | 없음 | 방치형, 힐링 |
| **중간** | **미드코어** | **높음** | **낮음** | **수집형 RPG, 가챠** |
| 높음 | 미드~하드코어 | 높음 | 중간 | 전략 RPG, 액션 RPG |
| 매우 높음 | 하드코어 | 중간 (소수 충성) | 높음 | ARPG, 소울라이크 |

### 3-4. 스킬 복잡도 설계 가이드

**단순 스킬의 예시 (나쁜 예)**
```
[강타]: 적 1명에게 공격력의 150% 피해를 입힙니다.
[강타+]: 적 1명에게 공격력의 200% 피해를 입힙니다.
[강타++]: 적 1명에게 공격력의 250% 피해를 입힙니다.
→ 이건 스킬 차별화가 아니라 수치 인플레이션임다
```

**적절한 복잡도의 예시 (좋은 예)**
```
에픽세븐 케이론 S1:
[공허의 일격]: 적 1명을 공격. 대상에게 버프가 있을 경우
               모든 적을 공격으로 변환. (조건부 범위 변환)
→ 스킬 설명 2줄, 조건 1개, 하지만 전략적으로 깊음
→ "버프 걸린 적이 있으면 전체 공격" = 상대방 구성을 읽어야 함
```

## 4. 등급간 차별화: 5가지 패턴 분석

### 4-1. 패턴 분류표

| 패턴 | 설명 | 대표 게임 | 유저 과금 동기 | 권고 여부 |
|------|------|----------|--------------|---------|
| **A. 수치 차이형** | 같은 스킬, 높은 등급이 수치만 더 높음 | 소녀전선, Idle Heroes, 레이든 블레이드 | 약함 (강해지기 위해 과금) | 비권고 |
| **B. 신규 효과 추가형** | 등급 오르면 기존 스킬에 새 효과 추가 | 원신 Constellation, 에픽세븐 EE, Summoners War 각성 | 강함 (이 캐릭터를 더 파고 싶음) | 권고 |
| **C. 스킬 수 차별화형** | 높은 등급일수록 스킬 슬롯 자체 증가 | 아크나이츠 (6★=3스킬, 4★=2스킬) | 매우 강함 (눈에 보이는 차이) | 강권고 |
| **D. 단계별 질적 변화형** | 강화 단계마다 스킬 효과가 질적으로 달라짐 | SWGoH (Omega→Zeta→Omicron), Summoners War 2차 각성 | 매우 강함 (단계별 목표 부여) | 강권고 |
| **E. 완전 리워크형** | 각성/진화 시 스킬셋 완전 교체 | AFK Arena Celestial/Hypogean, FF Brave Exvius NV | 강함 (수집의 판타지) | 상황에 따라 권고 |

### 4-2. 패턴별 상세 분석

#### 패턴 B — 신규 효과 추가형 (권고)

```
[원신 Constellation 사례]
라이덴 쇼군 (5★)
- C0 (기본): 원소 폭발 발동 후 번개 효과 부여
- C2 (2중 Constellation): 폭발 사용 후 에너지 회복 불가 시간 감소 → 연속 폭발 가능
- C6 (최고 Constellation): 칼 공격이 방어력 무시 + 추가 번개 피해
→ 같은 캐릭터인데 C0과 C6는 완전히 다른 전투 스타일

[에픽세븐 Exclusive Equipment 사례]
EE 장착 전: S3가 단순 단일 강타
EE 장착 후: S3 발동 시 추가 효과 추가 (캐릭터마다 다른 고유 효과)
→ "이 캐릭터의 EE를 언제 얻나"가 중기 목표가 됨
```

#### 패턴 C — 스킬 수 차별화형 (강권고)

```
[아크나이츠 등급별 스킬 수]
1★~2★: 스킬 없음 (자동 공격만)
3★:     스킬 1개
4★:     스킬 1개 (승격 후 스킬 2개)
5★:     스킬 2개 (승격 후 유지)
6★:     스킬 2개 (2차 승격 후 스킬 3개)

→ "6★ 오퍼레이터 = 스킬 3개" 라는 직관적 가치 전달
→ 뽑고 싶은 이유가 수치가 아니라 "더 많은 전략 옵션"임다
```

#### 패턴 D — 단계별 질적 변화형 (강권고)

```
[Star Wars: Galaxy of Heroes 업그레이드 계층]
Omega (Lv6→7): 스킬 수치 강화 (20~30% 증가)
Zeta (Lv7→8):  스킬에 완전히 새로운 효과 추가
               예: "아군 처치 시 보호막" → "처치 시 보호막 + 이 캐릭터 즉시 턴 획득"
Omicron (Lv8+): 특정 컨텐츠 전용 추가 효과 (Territory Battle 등에서만 발동)

→ 유저는 "Zeta를 어떤 캐릭터에게 먼저 줄 것인가"를 전략적으로 고민
→ 과금 포인트가 "강화 재료"가 아니라 "의사결정"에 있어 허들 낮음
```

### 4-3. 결론: 등급 차별화 설계 원칙

```
❌ 피해야 할 것: 수치만 올리는 등급 차별화
   (모든 등급이 같은 스킬, 수치만 다름 = 과금해도 "그냥 강해졌다"는 느낌)

✅ 해야 할 것: 등급 오를 때 새로운 "가능성"이 열려야 함
   새로운 스킬 효과 추가 OR 스킬 수 증가 OR 메커니즘 변화
   → "이 캐릭터 6성 만들면 무엇이 달라지나?" 에 명확한 답이 있어야 함

✅ 최우수 모델: 아크나이츠 (스킬 수 차별화) + SWGoH (3단계 질적 변화)의 조합
```

## 5. 스킬 간 차별화: 메커니즘 다양성의 기준

### 5-1. 스킬 간 차별화 수준 측정 기준

**차별화 점수 산정**:
- 같은 게임 내 캐릭터 10명을 선택
- 스킬 설명에서 고유한 메커니즘 키워드 수를 카운트
- 동일/유사 메커니즘 반복 비율을 역산

### 5-2. 장르별 스킬 간 차별화 수준

| 게임 | 차별화 수준 | 고유 메커니즘 예시 |
|------|-----------|----------------|
| 아크나이츠 | ★★★★★ | SP 회복 타입 3종 × 지속/순간/패시브 × 타깃 배치 연계 |
| 에픽세븐 | ★★★★★ | 소울번, 전투준비도 조작, 부활, 조건부 범위 전환, 무적 등 30+ 메커니즘 |
| Summoners War | ★★★★☆ | 버프/디버프 40+ 종류, 각성 전후 완전히 다른 스킬 |
| 원신 | ★★★★★ | 7원소 × 5반응 × 캐릭터별 원소 메커니즘 |
| Slay the Spire | ★★★★★ | ~75장 카드 중 동일 메커니즘 카드 없음 |
| Diablo IV | ★★★★☆ | 클래스별 14개 스킬 × Enhanced+2갈래 분기 = 42개 변형 |
| Fire Emblem Heroes | ★★★★★ | 패시브 A/B/C 각 수십 종 × 스킬 상속으로 무한 조합 |
| 블루 아카이브 | ★★★☆☆ | EX/Normal/Passive/Sub 4타입 구분, 캐릭터별 수치 차이 |
| Idle Heroes | ★★☆☆☆ | 대부분 "X% 피해 + 상태이상" 반복 구조 |
| Magic ManaStrike | ★★☆☆☆ | (서비스 종료) 메커니즘 다양성 부족 + 타깃 미스매치 |

### 5-3. 스킬 메커니즘 분류 사전 (설계 도구)

아래 카테고리를 참고해 스킬 간 차별화를 설계한다.

#### A. 타깃 타입
```
단일 / 전체 / 전방열 / 후방열 / 랜덤 N명 / 특정 상태 대상
조건부 전환 (에픽세븐: 버프 있으면 전체 공격)
```

#### B. 발동 트리거
```
수동 발동 / 쿨다운 자동 / 패시브 상시 / 조건부 (HP 임계값, 상태이상, 처치 후)
서포트 발동 (아군 공격 시 함께) / 반격 발동 / 사망 시 발동
```

#### C. SP/자원 관리
```
SP 생성형 / SP 소비형 / 에너지 충전형 / 쿨다운형 / 무제한형
아크나이츠의 SP 회복 타입: Auto(자동) / Offensive(공격) / Defensive(피격)
```

#### D. 상태 이상 / 버프 디버프
```
[디버프 카테고리]
즉발 피해: 출혈, 독, 화상 (DoT)
제어: 기절, 마비, 빙결, 공포, 수면
능력 감소: 공격력 감소, 방어력 감소, 회피율 감소
특수: 도발(타깃 강제), 봉인(스킬 사용 불가), 표식(피해 증폭)

[버프 카테고리]
즉발 회복: 즉시 HP, 쉴드
재생형: 턴당 HP
능력 증가: 공격력, 방어력, 속도, 크리티컬
특수: 무적, 전투준비도 조작, 추가 행동
```

#### E. 공간/배치 메커니즘
```
포지션 기반 (Darkest Dungeon: 스킬마다 사용 가능 열 다름)
사거리 기반 (아크나이츠: 근거리/원거리/드론)
지형 기반 (원신: 필드 설치 스킬, 벽 생성)
```

#### F. 연계/콤보 메커니즘
```
체인 공격 (아군이 연속 공격 시 보너스)
1 More 연쇄 (Persona 5: 약점 공격 → 추가 행동)
Burst Chain (NIKKE: Burst I→II→III 순서)
더블/트리플 테크 (Chrono Trigger: 2~3인 조합)
카드 합치기 (7DS: 동일 카드 합쳐 강화)
```

### 5-4. 차별화 설계 체크리스트

게임 내 스킬 10종을 나열했을 때, 아래 항목에서 각 스킬이 달라야 한다:

```
□ 타깃 타입이 다른가? (단일 vs 전체 vs 조건부)
□ 발동 트리거가 다른가? (수동 vs 패시브 vs 조건부)
□ 상태이상 카테고리가 다른가? (제어계 vs DoT vs 디버프)
□ 자원 구조가 다른가? (SP 소비 vs 충전형)
□ 전투 역할이 다른가? (딜 vs 탱크 vs 서포트 vs 컨트롤)

10종 중 동일 카테고리 스킬이 3개 이상 겹치면 차별화 실패
```

## 6. 스킬 성장 구조: 검증된 모델 6가지

### 6-1. 모델 비교표

| 모델 | 대표 게임 | 성장 느낌 | BM 연동 | 복잡도 | 권고 여부 |
|------|----------|---------|---------|--------|---------|
| **A. 레벨업형** | 블루 아카이브, Summoners War, 원신 | 점진적 수치 성장 | 재료 판매 | 낮음 | 기본형 |
| **B. 각성/승급형** | 아크나이츠 Mastery, 에픽세븐 각성 | 질적 도약 | 희귀 재료 | 중간 | 강권고 |
| **C. 단계별 해금형** | SWGoH Omega→Zeta→Omicron | 단계마다 새 목표 | 전용 재료 | 낮음 | 강권고 |
| **D. 스킬 상속형** | Fire Emblem Heroes, Persona 5 | 조합의 재미 | 희생 시스템 | 높음 | 심화 요소 |
| **E. 빌드 선택형** | Diablo IV, Last Epoch | 정체성 선택 | 없음 (BM 약함) | 중간 | PvE 특화 |
| **F. 덱빌딩형** | Slay the Spire, Monster Train | 발견의 재미 | 매우 낮음 | 높음 | 로그라이크 특화 |

### 6-2. 권장 모델 A+B+C 조합 (모바일 RPG 표준)

```
기본 성장 (Model A — 레벨업):
  스킬 Lv 1~10, 레벨업 시 수치 증가
  → 저가 재료로 꾸준한 성장감 제공

중기 성장 (Model B — 각성/승급):
  특정 레벨 도달 시 질적 변화 발생
  예: 스킬 Lv 5 → 효과 텍스트 변경, 신규 조건 추가
  → 중가 희귀 재료 소모, 수집 목표 제공

장기 성장 (Model C — 단계별 해금):
  최고 등급 도달 후 추가 스킬 특화 단계
  예: Mastery 0→1→2→3 (각 24시간 소요, 희귀 재료)
  → 하드코어 유저의 최종 목표, 과금 포인트

결과: 단기/중기/장기 목표가 모두 스킬 시스템 내에 존재
```

### 6-3. 스킬 성장 관련 핵심 실수

```
❌ 실수 1: 모든 스킬을 동일한 재료로 업그레이드
   → 어느 스킬을 먼저 올릴지 고민이 없어짐 (전략적 선택 제거)
   → 해결책: 스킬 타입별 전용 재료 부여 (아크나이츠처럼)

❌ 실수 2: 스킬 레벨업 효과가 수치만 오름
   → 레벨 1→10 동안 텍스트가 바뀌지 않으면 성장감 없음
   → 해결책: 중간 단계에서 반드시 질적 변화 1회 이상

❌ 실수 3: 스킬 성장 재료가 너무 쉽게 구해짐
   → 콘텐츠 소비 속도가 빨라져 지속성 손상
   → 해결책: 최종 Mastery 재료는 가장 어려운 콘텐츠에서만 획득
```

## 7. 제약 설계: 전략적 깊이의 원천

### 7-1. "제약이 전략을 만든다" — 데이터 증명

성공한 게임들은 예외 없이 **의도적 제약**을 설계했다.

| 제약 유형 | 게임 사례 | 전략적 효과 |
|----------|----------|-----------|
| **슬롯 수 제한** | 포켓몬 4개, Darkest Dungeon 7→4, Tactics Ogre 4개 | "무엇을 포기하느냐" 고뇌 생성 |
| **자원 제한** | BG3 스펠 슬롯, STS 에너지 3점, FGO 쿨다운 | "이 턴에 무엇을 써야 하나" 긴장감 |
| **포지션 제한** | Darkest Dungeon 열 제약, 아크나이츠 배치 제약 | 배치 자체가 전략 |
| **타입/상성 제한** | 포켓몬 18타입 상성, 원신 원소 반응, Persona 약점 | 상대를 파악해야 최적화 가능 |
| **런 내 변동성** | Slay the Spire 덱 드래프트, Hades Boon 선택 | 매 플레이 다른 빌드 = 재플레이 동기 |

### 7-2. 모바일 RPG에 적합한 제약 설계 권고

```
권고 제약 1: 스킬 슬롯 수 고정 (4~5개)
  모든 스킬을 사용할 수 없게 제한
  → "어떤 스킬을 장착할 것인가" 결정이 전략
  → 적용 사례: 아크나이츠 스킬 선택, Darkest Dungeon 4/7

권고 제약 2: SP/에너지 자원 관리
  강력한 스킬에 명확한 비용 부과
  → "언제 궁극기를 쓸 것인가" 타이밍 전략
  → 적용 사례: NIKKE 버스트 게이지, 에픽세븐 소울번

권고 제약 3: 팀 구성 제약
  파티 내 같은 역할 중복 시 패널티 or 다양성 시 보너스
  → 무지성 최강 스킬 5개 스택을 방지
  → 적용 사례: FGO 클래스 상성, NIKKE Burst I/II/III 체인

권고 제약 4: 포지션/배치 제약
  스킬의 효과가 포지션에 따라 달라짐
  → 팀 배치 자체가 전략 콘텐츠화
  → 적용 사례: Darkest Dungeon, 아크나이츠, Brown Dust 2
```

### 7-3. 과도한 제약의 실패 사례

```
❌ 과도한 제약 사례: Path of Exile (모바일 버전)
   스킬 링킹 시스템의 복잡도가 너무 높아 신규 유저 이탈
   → 강력하지만 모바일 타깃에는 적합하지 않음

❌ 제약 없는 실패 사례: 뮤 오리진2
   레벨업 시 자동으로 최강 스킬이 해금
   → 유저의 선택이 없어 전략 깊이 제로
   → 스킬이 아니라 과금력이 전부인 구조
```

## 8. 시너지 시스템: 발견 재미의 설계

### 8-1. 시너지 유형 분류

| 유형 | 설명 | 대표 사례 | 효과 강도 |
|------|------|----------|---------|
| **스킬-스킬 시너지** | 스킬 A 사용 후 스킬 B 효과 증폭 | 에픽세븐 체인, Persona 1 More | ★★★★★ |
| **캐릭터-캐릭터 시너지** | 특정 캐릭터 조합 시 추가 효과 | 원신 공명, NIKKE 버스트 체인 | ★★★★☆ |
| **스킬-상태이상 시너지** | 특정 상태이상에 반응하는 스킬 | 에픽세븐, Darkest Dungeon | ★★★★★ |
| **원소/타입 반응** | 두 속성 충돌로 새 효과 발생 | 원신 원소 반응, 포켓몬 타입 상성 | ★★★★★ |
| **파티 패시브 시너지** | 미배치 캐릭터 패시브가 전체에 영향 | FEH Z어빌리티, Brown Dust 2 | ★★★★☆ |
| **콤보 기술** | N명 조합 시 고유 스킬 해금 | Chrono Trigger 더블/트리플 테크 | ★★★★☆ |

### 8-2. 시너지 설계 황금 법칙

```
법칙 1: 시너지는 "발견"이 가능해야 한다
  → 튜토리얼에서 모두 알려주면 발견 재미가 없음
  → 일부 시너지는 유저가 직접 시험해봐야 알게 설계

법칙 2: 시너지는 "조합"이지 "정답"이 아니어야 한다
  → A+B = 최강이면 A+B만 쓰게 됨
  → A+B, C+D, E+F 각각이 다른 상황에서 유리해야 함

법칙 3: 시너지의 계층 구조
  기본 시너지: 같은 속성 조합 → 약간의 보너스
  중급 시너지: 조건 성립 시 대형 보너스 (상태이상+스킬)
  최상급 시너지: 특정 3~4명 조합에서만 발동하는 콤보 기술

법칙 4: 시너지 발견이 소셜 콘텐츠가 된다
  → 원신의 "원소 반응 공략", 에픽세븐의 "시너지 팀 공유"
  → 커뮤니티가 시너지를 연구하고 공유하게 유도
```

### 8-3. 최우수 시너지 시스템 분석

#### 원신: 원소 반응 시스템

```
7원소 × 5반응 = 35가지 반응 (실제 조합 수)

불+물 = 기화 (피해 1.5~2배)
불+번개 = 과부하 (범위 피해)
물+번개 = 감전 (지속 피해)
얼음+물 = 빙결 (이동 불가)
바람+모든 원소 = 확산 (원소 퍼뜨림)

→ 팀 구성 = "어떤 반응 루프를 만들 것인가"의 문제
→ 같은 원소라도 조합 순서에 따라 효과 다름
→ 결과: 팀 빌딩 = 게임의 핵심 전략 콘텐츠
```

#### Persona 5 Royal: 1 More 연쇄

```
약점 공격 or 크리티컬 → 1 More (추가 행동)
                         ↓
              Baton Pass → 다음 아군에게 행동 넘김
                           공격력 ×1.1 → ×1.25 → ×1.5
                                         ↓
                      All-Out Attack 가능 시 파티 전원 공격

→ 약점을 파악하는 것이 전투의 핵심 기술
→ 3연속 Baton Pass 달성 = "완벽한 플레이"의 성취감
→ 3번의 선택 × 성취감 × 시각적 연출 = 몰입도 극대화
```

## 9. 실패 패턴 분석

### 9-1. 스킬 시스템으로 인한 실패 사례

| 게임 | 실패 원인 | 교훈 |
|------|----------|------|
| **Magic ManaStrike** (서비스 종료) | MtG IP + Clash Royale 구조 혼합. MtG 팬은 카드게임 복잡도 원하고, 모바일 유저는 단순함 원함 → 타깃 미스매치 | IP 기반 스킬 시스템 설계 전 타깃 유저 명확화 필수 |
| **Idle Heroes** | 스킬 간 메커니즘 차별화 부족, "X% 피해 + 상태이상" 반복 | 스킬 차별화 없으면 컬렉션 동기가 스킬이 아닌 순수 수치 경쟁으로 변질 |
| **Dislyte** | 스킬 시스템 평범, 아트/음악 퀄리티만으로 론칭 초기 화제 → 핵심 게임플레이 약해 이탈 | 아트가 어텐션, 스킬이 리텐션임다 |
| **뮤 오리진2** | 레벨업 자동 해금, 유저 선택 없음, 스킬 = 수치 경쟁 | 선택이 없는 성장 = 게임이 아니라 과금 시뮬레이터 |
| **Dungeon Hunter 5** | 장비 교체=스킬 교체 방식이 독창적이나 스킬 간 연결 깊이 부족 | 독창적 메커니즘도 시너지 없으면 얕게 느껴짐 |

### 9-2. 스킬 설계 7대 안티패턴

```
안티패턴 1: 숫자만 다른 스킬
  "강타 100%" "강타+ 150%" "강타++ 200%"
  → 이건 스킬이 아니라 수치 테이블임다

안티패턴 2: 모든 캐릭터가 같은 스킬 구조
  "모든 캐릭터: 공격1 + 범위공격 + 궁극기 + 패시브"
  → 캐릭터 고유성 없음 = 수집 동기 손상

안티패턴 3: 등급이 올라도 스킬이 바뀌지 않음
  6성이 1성보다 100% 강하지만 스킬 텍스트가 동일
  → "강함을 사는 것" 이지 "다름을 사는 것"이 아님

안티패턴 4: 최강 스킬 조합이 단 하나
  "이 팀 구성이 정답" = 나머지 캐릭터 쓰레기화
  → FotM(Flavor of the Month) = 메타 고착 = 이탈률 증가

안티패턴 5: 스킬 설명이 너무 복잡하거나 모호함
  조건이 5개 이상인 스킬 = 유저가 직접 테스트해봐야 이해
  → 단, 발견 재미를 위한 적절한 모호함은 OK

안티패턴 6: 스킬 성장 재료가 획일적
  모든 스킬을 같은 재료로 업그레이드 = 전략 선택 없음
  → 스킬 타입별 전용 재료 설계 권고

안티패턴 7: 시너지 없는 개인기 나열
  각 스킬이 독립적으로만 작동 = 조합 탐구 재미 없음
  → 최소 스킬 2개 이상이 서로 연계되도록 설계
```

## 11. 모바일 RPG 설계 권고안 (핵심 결론)

### 11-1. 스킬 설계 황금 공식

100종 게임 분석을 통해 도출한 모바일 RPG 스킬 설계의 황금 공식임다.

```
[스킬 수]
액티브 스킬: 4개 (황금 구간)
패시브 스킬: 1~2개
총 스킬 수: 5~6개

[스킬 복잡도]
미드코어 타깃: 중간 복잡도 (스킬 설명 2~4줄, 조건부 효과 1~2개)
하드코어 타깃: 높은 복잡도 (조건부 효과 3개 이상, 상태이상 연계)

[스킬 간 차별화]
4개 액티브 스킬이 각각 다른 타깃 타입 + 다른 발동 트리거
최소 2개 스킬이 서로 시너지를 가져야 함

[등급 차별화]
기본: 수치 차이 (20~30% 범위)
중요: 등급 오를 때 신규 효과 추가 (텍스트 변경 발생)
최우수: 특정 등급 도달 시 스킬 수 자체 증가

[스킬 성장]
단기: Lv 1~최대 레벨업 (수치 증가)
중기: 각성/승급 시 질적 변화 (반드시 1~2회)
장기: Mastery/특화 단계 (희귀 재료, 하드코어 목표)
```

### 11-2. 검증된 체크리스트 (기획안 완성 전 필수)

```
스킬 구조 체크
□ 캐릭터당 스킬 수가 4~6개인가?
□ 각 스킬이 서로 다른 메커니즘을 가지는가?
□ 스킬 설명 텍스트가 2~4줄 범위인가?
□ 적어도 2개 스킬이 서로 시너지를 가지는가?

등급 차별화 체크
□ 높은 등급 캐릭터가 단순히 "더 강한" 게 아닌가?
□ 등급 오를 때 스킬 텍스트가 바뀌는가?
□ "이 캐릭터를 고등급으로 키우면 무엇이 달라지나?"에 명확한 답이 있는가?

스킬 성장 체크
□ 스킬 레벨업이 수치만 오르는 구간이 Lv1~10 중 7 이상인가? (개선 필요)
□ 최소 1번은 레벨업 시 질적 변화(텍스트 변경)가 있는가?
□ 장기 목표(Mastery/특화) 시스템이 있는가?

차별화 체크
□ 동일 게임 내 같은 메커니즘을 반복하는 스킬이 3개 이상인가? (개선 필요)
□ 각 캐릭터가 "교체할 수 없는 고유 역할"을 가지는가?
□ 시너지 팀 조합이 5가지 이상 존재하는가?

제약 체크
□ 유저가 "무엇을 포기할 것인가" 고민하는 순간이 있는가?
□ 강력한 스킬에 명확한 비용/쿨다운이 있는가?
□ "정답 팀"이 1개만 존재하지 않는가?
```

### 11-3. 최종 한 줄 요약

> **"스킬 수는 4~5개, 각 스킬은 다른 메커니즘, 등급 오르면 새로운 가능성이 열려야 한다.
> 그리고 제약이 전략을 만든다 — 모든 스킬을 쓸 수 있으면 전략이 없다."**
